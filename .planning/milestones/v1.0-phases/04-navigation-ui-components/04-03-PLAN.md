**ButtonVariant enum:**
```kotlin
enum class ButtonVariant { Default, Secondary, Ghost, Destructive }
```

**TerminalButton composable:**
```kotlin
@Composable
fun TerminalButton(
    text: String,
    onClick: () -> Unit,
    modifier: Modifier = Modifier,
    variant: ButtonVariant = ButtonVariant.Default,
    enabled: Boolean = true,
    icon: (@Composable () -> Unit)? = null,
)
```

Implementation requirements:
- Read ALL colors from `TerminalTheme.colors` -- no hardcoded hex values
- Read corner radius from `TerminalTheme.radius` (use .sm for buttons based on Pencil)
- Read typography from `TerminalTheme.typography` (use .sm for button text)
- Read spacing/gap from `TerminalTheme.spacing` / `TerminalTheme.gap`
- Read border width from `TerminalTheme.borders`

Variant color mapping (verify against .pen file):
- Default: bg=accent, text=surface (white text on green)
- Secondary: bg=surface, text=text, border=border
- Ghost: bg=transparent, text=textMuted, no border
- Destructive: bg=errorBg, text=error

Disabled state: apply `TerminalTheme.opacity.medium` via `Modifier.alpha()`

Use Foundation primitives ONLY:
- `Box` with `Modifier.clickable(enabled, onClick = onClick)` for the clickable area
- `Modifier.clip(RoundedCornerShape(...))` for corner radius
- `Modifier.background(backgroundColor)` for fill
- `Modifier.border(width, color, shape)` for borders (only Secondary variant)
- `Row` with `Arrangement.spacedBy(gap.sm)` for icon + text layout
- `BasicText` for the label

Do NOT use Material3 `Button`, `TextButton`, `OutlinedButton` or any compose.material3 import.

- Error state: border color changes to error, optional errorMessage shown below in error color

- All colors from TerminalTheme.colors

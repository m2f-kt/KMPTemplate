    2. Create `OAuthService.kt`:
       - `handleGoogleCallback(principal: OAuthAccessTokenResponse.OAuth2): AuthResponse` -- extracts email from Google userinfo endpoint, creates user if not exists (via UserRepository), generates token pair, returns AuthResponse
       - `handleAppleCallback(principal: OAuthAccessTokenResponse.OAuth2): AuthResponse` -- similar for Apple (decodes id_token for email)
       - Both use existing UserRepository.insert() and JwtTokenProvider.generateTokenPair()
       - If user exists by email, log them in instead of creating duplicate

    3. Create `OAuthRoutes.kt`:
       - `fun Route.oauthRoutes(oauthService: OAuthService)` with:
         - `authenticate("google-oauth") { get("/api/auth/oauth/google") { /* Ktor redirects */ } }`
         - `get("/api/auth/oauth/google/callback") { /* handle callback */ }`
         - Same pattern for Apple with `authenticate("apple-oauth")`

    4. Install OAuth authentication providers in Application.kt:
       - `install(Authentication) { oauth("google-oauth") { ... } }` using Ktor's OAuthServerSettings.OAuth2ServerSettings
       - Google: authorizeUrl = "https://accounts.google.com/o/oauth2/v2/auth", accessTokenUrl = "https://oauth2.googleapis.com/token", scopes = ["openid", "profile", "email"]
       - Apple: authorizeUrl = "https://appleid.apple.com/auth/authorize", accessTokenUrl = "https://appleid.apple.com/auth/token", scopes = ["name", "email"]
       - Add `oauthRoutes(oauthService)` to routing block

    **Password Reset:**

    5. Create `PasswordResetTokensTable.kt`: id (uuid), userId (uuid ref), tokenHash (varchar), expiresAt (datetime), used (boolean default false)

       - `resetPassword(request: ResetPasswordRequest)`: validates token, updates password hash via UserRepository, marks token used
       - Uses Arrow Raise<DomainError> pattern consistent with AuthService

    8. Add password reset routes to AuthRoutes.kt:

Fix dashboard sidebar navigation to use state-based content switching instead of top-level route navigation. Currently, clicking sidebar nav items (or mobile bottom tabs) navigates to entirely new screens, replacing the dashboard (and its sidebar) with standalone PlaceholderScreens. The fix changes the architecture so the sidebar/bottom nav stays persistent and only the content area swaps.

Additionally, embed profile within the dashboard shell on desktop (so the sidebar remains visible) and add back navigation to the profile. Ensure all content areas are scrollable.

Purpose: UAT Tests 7 and 10 both failed because the sidebar disappears when navigating to sub-sections. This is the primary UX gap in Phase 5.
Output: DashboardScreen with state-driven content switching, profile embedded in dashboard shell, all placeholder routes removed from top-level navigation.
</objective>

    **AppNavHost.kt:**
    - Remove the four `composable<ProcessesRoute>`, `composable<LogsRoute>`, `composable<DeploymentsRoute>`, `composable<SettingsRoute>` blocks entirely (lines ~197-231).
    - Remove the `PlaceholderScreen` private composable at the bottom of the file (lines ~235-289). It is no longer used.
    - Update the `composable<DashboardRoute>` block:
      - Remove the four navigation callback params (onNavigateToProcesses, onNavigateToLogs, onNavigateToDeployments, onNavigateToSettings).
      - Add: inject `koinViewModel<ProfileViewModel>()` and collect its state.
      - Add: pass `profileContent` lambda to DashboardScreen that renders ProfileScreen with all its callbacks wired to the ProfileViewModel.
      - Add: `onShowProfile = { dashboardViewModel.showProfile() }` and `onHideProfile = { dashboardViewModel.hideProfile() }`.
      - Handle logout from within profile: the ProfileViewModel's logoutTriggered flag should be observed with a LaunchedEffect inside the DashboardRoute composable block, navigating to LoginRoute with popUpTo(0) { inclusive = true } on trigger.

    The DashboardRoute composable block should look approximately like:
    ```kotlin
    composable<DashboardRoute> {
        val dashboardViewModel = koinViewModel<DashboardViewModel>()
        val dashboardState by dashboardViewModel.state.collectAsStateWithLifecycle()
        val profileViewModel = koinViewModel<ProfileViewModel>()
        val profileState by profileViewModel.state.collectAsStateWithLifecycle()

        DashboardScreen(
            state = dashboardState,
            onNavItemSelected = dashboardViewModel::selectNavItem,
            onShowProfile = { dashboardViewModel.showProfile() },
            onHideProfile = { dashboardViewModel.hideProfile() },
            onLogout = {
                navController.navigate(LoginRoute) {
                    popUpTo(0) { inclusive = true }
                }
            },
            profileContent = {
                ProfileScreen(
                    state = profileState,
                    onStartEditing = profileViewModel::startEditing,
                    onCancelEditing = profileViewModel::cancelEditing,
                    onEditNameChange = profileViewModel::onEditNameChange,
                    onEditEmailChange = profileViewModel::onEditEmailChange,
                    onSaveProfile = profileViewModel::saveProfile,
                    onLogout = profileViewModel::logout,
                    onBack = { dashboardViewModel.hideProfile() },
                )
            },
        )

        // Handle profile logout
        LaunchedEffect(profileState.logoutTriggered) {
            if (profileState.logoutTriggered) {
                navController.navigate(LoginRoute) {
                    popUpTo(0) { inclusive = true }
                }
            }
        }
    }
    ```

    - Keep the standalone `composable<ProfileRoute>` block as a fallback/deep-link route. This ensures that if anything still navigates to ProfileRoute directly (unlikely after this change), it still works. But the primary profile access path is now through the dashboard shell.
    - Remove unused imports for ProcessesRoute, LogsRoute, DeploymentsRoute, SettingsRoute if the compiler flags them.

  <files>

  <action>
    Create `setup.sh` at project root with `#!/usr/bin/env bash` and `set -euo pipefail`.

    **Terminal aesthetic header** (matching the project's terminal design):
    ```
    echo -e "${GREEN}> terminal setup_project${NC}"
    echo "// Project configuration wizard"
    echo ""
    ```

    **Interactive prompts:**
    1. `read -p "Project name (e.g., MyApp): " PROJECT_NAME`
    2. `read -p "Package name (e.g., com.company.app): " PACKAGE_NAME`
    3. `read -p "Database name (e.g., myapp_db): " DB_NAME`

    **Input validation (before any modifications):**
    - PROJECT_NAME: non-empty, alphanumeric + spaces/hyphens/underscores
    - PACKAGE_NAME: must match `^[a-z][a-z0-9]*(\.[a-z][a-z0-9]*)+$`
    - DB_NAME: non-empty, alphanumeric + underscores, no spaces
    - Exit with descriptive error message if validation fails

    **Derived values:**
    ```bash
    PACKAGE_PATH="${PACKAGE_NAME//./\/}"
    OLD_PACKAGE="com.m2f.template"
    OLD_PACKAGE_PATH="com/m2f/template"
    OLD_SERVER_PACKAGE="com.m2f"
    OLD_SERVER_PATH="com/m2f"
    PROJECT_NAME_LOWER=$(echo "$PROJECT_NAME" | tr '[:upper:]' '[:lower:]' | tr ' ' '-')
    ```

    **Dry-run summary (show before executing):**
    ```
    echo ""
    echo "status: preview"
    echo "  project_name: $PROJECT_NAME"
    echo "  package:      $OLD_PACKAGE -> $PACKAGE_NAME"
    echo "  database:     template -> $DB_NAME"
    echo "  root_project: template -> $PROJECT_NAME_LOWER"
    echo ""
    read -p "Proceed? (y/N): " CONFIRM
    [[ "$CONFIRM" != "y" && "$CONFIRM" != "Y" ]] && echo "Aborted." && exit 0
    ```

    **Rename operations (in order):**

    1. **Update package references in all Kotlin files (.kt):**
       `find . -name "*.kt" -not -path "./.gradle/*" -not -path "./build/*" -not -path "./.planning/*" -exec sed -i '' "s/$OLD_PACKAGE/$PACKAGE_NAME/g" {} +`
       Also rename `com.m2f.server` -> derived server package, `com.m2f.core` -> derived core package.

    2. **Update package references in Gradle files (.kts):**
       `find . -name "*.gradle.kts" -not -path "./.gradle/*" -exec sed -i '' "s/$OLD_PACKAGE/$PACKAGE_NAME/g" {} +`
       This covers namespace, applicationId, and package references in build scripts.

    3. **Update settings.gradle.kts:**
       `sed -i '' "s/rootProject.name = \"template\"/rootProject.name = \"$PROJECT_NAME_LOWER\"/g" settings.gradle.kts`

    4. **Update docker-compose.yml database name:**
       `sed -i '' "s/POSTGRES_DB: template/POSTGRES_DB: $DB_NAME/g" docker-compose.yml`

    5. **Update server database configuration** (DataSource.kt or Env.kt):
       `sed -i '' "s/database = \"template\"/database = \"$DB_NAME\"/g"` on relevant config files.

    6. **Move source directories to new package path:**
       For each source set directory containing `com/m2f/template`:
       - Create new directory tree matching PACKAGE_PATH
       - Move *.kt files from old path to new path
       - Remove empty old directories
       Handle all module source sets: composeApp (commonMain, androidMain, iosMain, jvmMain, wasmJsMain), shared, core/models, core/sdk, core/storage, app/auth, app/dashboard, app/designsystem, server, server/auth, server/core/config, server/core/database, server/core/security

    7. **Handle server package (com.m2f.core, com.m2f.server):**
       These have a different package root. Map them to the new package's server/core equivalents:
       - `com.m2f.core` -> `{first_two_segments}.core` (e.g., com.company.core)
       - `com.m2f.server` -> `{first_two_segments}.server` (e.g., com.company.server)
       Move server source directories accordingly.

    8. **Update AndroidManifest.xml** if it contains package references.

    9. **Update .iml files** or delete them (IDE will regenerate).

    **Post-rename verification:**
    ```bash
    echo ""
    echo "status: verifying..."
    # Check no old package references remain
    REMAINING=$(grep -r "com.m2f.template" --include="*.kt" --include="*.kts" -l . 2>/dev/null | grep -v ".gradle/" | grep -v "build/" | grep -v ".planning/" || true)
    if [[ -n "$REMAINING" ]]; then
        echo "warning: old package references found in:"
        echo "$REMAINING"
    else
        echo "status: package_rename complete"
    fi
    ```

    **Optional build verification:**
    ```bash
    echo ""
    read -p "Run build verification? (y/N): " BUILD_CHECK
    if [[ "$BUILD_CHECK" == "y" || "$BUILD_CHECK" == "Y" ]]; then
        echo "status: building..."
        ./gradlew build --no-daemon 2>&1 | tail -5
    fi
    ```

    **Completion message:**
    ```
    echo ""
    echo -e "${GREEN}> setup_complete${NC}"
    echo "  project: $PROJECT_NAME"
    echo "  package: $PACKAGE_NAME"
    echo "  database: $DB_NAME"
    echo ""
    echo "// Run './gradlew build' to verify"
    echo "// Run 'docker-compose up -d' to start the database"
    ```

    Make script executable: `chmod +x setup.sh`

    **Platform compatibility:** Use `sed -i ''` for macOS. Add detection for Linux (`sed -i`) vs macOS (`sed -i ''`) at script top:
    ```bash
    if [[ "$(uname)" == "Darwin" ]]; then
        SED_INPLACE="sed -i ''"
    else
        SED_INPLACE="sed -i"
    fi
    ```
    Then use `$SED_INPLACE` in all sed commands.
  </action>

**Create buildSrc directory and Gradle setup:**

1. Create `buildSrc/settings.gradle.kts`:
```kotlin
dependencyResolutionManagement {
    versionCatalogs {
        create("libs") {
            from(files("../gradle/libs.versions.toml"))
        }
    }
}
```

2. Create `buildSrc/build.gradle.kts`:
```kotlin
plugins {
    `kotlin-dsl`
}

repositories {
    google()
    mavenCentral()
    gradlePluginPortal()
    maven("https://maven.pkg.jetbrains.space/public/p/ktor/eap")
}

dependencies {
    implementation(libs.plugins.kotlinJvm.get().let { "${it.pluginId}:${it.pluginId}.gradle.plugin:${it.version}" })
    implementation(libs.plugins.kotlinx.serialization.get().let { "${it.pluginId}:${it.pluginId}.gradle.plugin:${it.version}" })
    implementation(libs.plugins.kover.get().let { "${it.pluginId}:${it.pluginId}.gradle.plugin:${it.version}" })
    implementation(libs.plugins.detekt.get().let { "${it.pluginId}:${it.pluginId}.gradle.plugin:${it.version}" })
}
```

Note: The exact dependency format for plugins in buildSrc may need adjustment. The pattern is `"plugin.id:plugin.id.gradle.plugin:version"`. Verify by checking each plugin's published artifact coordinates. For example:
- Kotlin JVM: `"org.jetbrains.kotlin.jvm:org.jetbrains.kotlin.jvm.gradle.plugin:2.2.10"`
- Serialization: `"org.jetbrains.kotlin.plugin.serialization:org.jetbrains.kotlin.plugin.serialization.gradle.plugin:2.2.10"`
- Kover: `"org.jetbrains.kotlinx.kover:org.jetbrains.kotlinx.kover.gradle.plugin:0.9.1"`
- Detekt: `"io.gitlab.arturbosch.detekt:io.gitlab.arturbosch.detekt.gradle.plugin:1.23.8"`

If the dynamic `libs.plugins` approach does not work in buildSrc (it sometimes doesn't due to classpath ordering), hardcode the dependency strings using the exact versions from libs.versions.toml.

3. Create `buildSrc/src/main/kotlin/server-module-convention.gradle.kts`:
```kotlin
plugins {
    id("org.jetbrains.kotlin.jvm")
    id("org.jetbrains.kotlin.plugin.serialization")
    id("org.jetbrains.kotlinx.kover")
    id("io.gitlab.arturbosch.detekt")
}

kotlin {
    compilerOptions {
        freeCompilerArgs.set(listOf("-Xcontext-parameters"))
    }
}
```

This replaces the duplicated `allprojects` blocks in `server/build.gradle.kts` and `server/core/build.gradle.kts`.

4. Create `buildSrc/src/main/kotlin/kmp-library-convention.gradle.kts`:
```kotlin
// Convention plugin for KMP library modules (core:models, core:sdk, core:storage)
// Applied by KMP modules that target Android, iOS, JVM, and WASM
// Each module still declares its own KMP targets and dependencies
// This plugin provides the shared compiler configuration

plugins {
    id("org.jetbrains.kotlin.multiplatform")
    id("org.jetbrains.kotlin.plugin.serialization")
}
```

This will be used by new KMP modules created in Plan 02.

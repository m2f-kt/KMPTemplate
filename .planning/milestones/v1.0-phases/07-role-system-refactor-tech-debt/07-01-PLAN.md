```kotlin
@Serializable(with = UserRoleSerializer::class)
sealed class UserRole {
    abstract val value: String
    abstract val level: Int

    data object User : UserRole() { override val value = "USER"; override val level = 0 }
    data object Admin : UserRole() { override val value = "ADMIN"; override val level = 1 }
    data object PowerAdmin : UserRole() { override val value = "POWER_ADMIN"; override val level = 2 }

    companion object {
        fun fromString(role: String): UserRole = when (role.uppercase()) {
            "USER" -> User
            "ADMIN" -> Admin
            "POWER_ADMIN", "POWERADMIN" -> PowerAdmin
            else -> User
        }
        val entries: List<UserRole> = listOf(User, Admin, PowerAdmin)
    }
}
```

Create a custom `UserRoleSerializer` (as a KSerializer in the same file or as an `object : KSerializer<UserRole>`) that serializes UserRole to/from its `value` string. This ensures flat string on the wire: `"role":"ADMIN"` NOT `"role":{"type":"ADMIN"}`. The serializer's `serialize` writes `encoder.encodeString(value.value)` and `deserialize` reads `UserRole.fromString(decoder.decodeString())`.

CRITICAL: The serializer ensures backward compatibility -- the JSON wire format stays as a plain string, identical to the current `role: String` format. Existing clients will not break.

Then update `UserDtos.kt`:

Do NOT add `@Serializable` with `@SerialName` on each variant -- use the custom serializer approach instead. The `@SerialName` approach would require polymorphic serialization setup and produces nested JSON.

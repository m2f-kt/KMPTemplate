<objective>
Add ktor-server-sse dependency, create the custom streaming strategy graph that fixes the chatAgentStrategy() infinite loop bug, and refactor ChatAgentService to use per-request AIAgent instances with callbackFlow for streaming.

Purpose: The current ChatAgentService uses `chatAgentStrategy()` which has a confirmed infinite loop bug -- when the LLM responds with plain text instead of tool calls, the strategy loops `giveFeedbackToCallTools` indefinitely. This plan replaces it with a custom strategy using `nodeLLMRequestStreaming()` that accepts text responses as valid completions and streams them incrementally. It also removes the now-unnecessary SayToUser/AskUser/ExitTool tools since streaming text IS the response.

Output: A refactored ChatAgentService with `streamChat()` returning `Flow<String>`, backed by a custom strategy graph, ready for SSE route wiring in Plan 02.
</objective>

**Step 3: Create `ChatStreamingStrategy.kt`.**

Create a new file `server/ai/src/main/kotlin/com/m2f/server/ai/agents/ChatStreamingStrategy.kt` with a function `chatStreamingStrategy()` that builds a custom strategy graph. This function takes a `processFrame: (String) -> Unit` callback parameter that receives text chunks as they stream.

The strategy graph should have these nodes:
1. `nodeLLMRequestStreaming()` -- sends the prompt and returns `Flow<StreamFrame>`
2. A custom `processFrames` node that iterates the `Flow<StreamFrame>`, calls `processFrame(frame.text)` for each `StreamFrame.Append`, and collects any `StreamFrame.ToolCall` frames into a list of `Message.Tool.Call`
3. `nodeExecuteMultipleTools(parallelTools = true)` -- executes tool calls if present
4. `nodeLLMSendMultipleToolResults()` -- sends tool results back to LLM

Edges:
- `nodeStart forwardTo nodeStreaming`
- `nodeStreaming forwardTo processFrames`
- `processFrames forwardTo executeMultipleTools onMultipleToolCalls { true }` (if tool calls present)
- `executeMultipleTools forwardTo sendToolResults`
- `sendToolResults forwardTo nodeStreaming` (loop back for more after tool results)
- `processFrames forwardTo nodeFinish onCondition { it.filterIsInstance<Message.Tool.Call>().isEmpty() }` (text-only response = done)

**5. Add `streamChat()` method returning `Flow<String>`:**
```kotlin
fun streamChat(userId: String, conversationId: String, input: String): Flow<String> =
    callbackFlow {
        val agentId = "user:$userId:conv:$conversationId"
        var agent: AIAgent<String, Any>? = null

        val strategy = chatStreamingStrategy { text ->
            trySend(text)
        }

        try {
            agent = AIAgent(
                id = agentId,
                promptExecutor = executor,
                agentConfig = agentConfig,
                strategy = strategy,
            ) {
                install(Persistence) {
                    storage = persistenceStorage
                    enableAutomaticPersistence = true
                    rollbackStrategy = RollbackStrategy.MessageHistoryOnly
                }
            }

            agent.run(input)
            close()
        } catch (e: Exception) {
            trySend("[ERROR] Agent failed: ${e.message}")
            close()
        }

        awaitClose {
            runBlocking {
                try { agent?.close() } catch (_: Exception) {}
            }
        }
    }
```

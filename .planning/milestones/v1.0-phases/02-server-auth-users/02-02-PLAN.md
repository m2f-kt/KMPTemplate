  <action>
    **Step 1: Add login to AuthService**
    Add `context(raise: Raise<DomainError>) suspend fun login(request: LoginRequest): AuthResponse` to AuthService:
    1. `val user = userRepository.findByEmail(request.email)` -- if null, `raise(InvalidCredentials())`. Do NOT say "user not found" -- use same generic InvalidCredentials to prevent user enumeration.
    2. `val passwordValid = passwordHasher.verify(request.password, user.passwordHash)` -- if false, `raise(InvalidCredentials())`
    3. Generate token pair: `val (authResponse, rawRefreshToken) = tokenProvider.generateTokenPair(user.id.toString(), user.role)`
    4. Hash and store refresh token: `val hashedToken = tokenProvider.hashRefreshToken(rawRefreshToken)` then `refreshTokenRepository.store(user.id, hashedToken, expiresAt)` where expiresAt is computed from `tokenProvider.getRefreshTokenExpiry()` added to current time
    5. Return authResponse

    **Step 2: Add login route to AuthRoutes**
    Inside the existing `route("/api/auth")` block in `AuthRoutes.kt`, add:
    ```kotlin
    post("/login") {
        conduit {
            val request: LoginRequest = getModel()
            authService.login(request)
        }
    }
    ```

    **Step 3: Add refresh to AuthService**
    Add `context(raise: Raise<DomainError>) suspend fun refresh(request: RefreshTokenRequest): AuthResponse` to AuthService:
    1. Hash the incoming refresh token: `val hashedToken = tokenProvider.hashRefreshToken(request.refreshToken)`
    2. Find valid token: `val tokenRecord = refreshTokenRepository.findValidToken(hashedToken)` -- if null, `raise(TokenInvalid())`
    3. Revoke old token: `refreshTokenRepository.revokeById(tokenRecord.id)` (refresh token rotation)
    4. Look up user for role: `val user = userRepository.findById(tokenRecord.userId)` -- if null (deleted user), `raise(TokenInvalid())`
    5. Generate new token pair: `val (authResponse, rawRefreshToken) = tokenProvider.generateTokenPair(user.id.toString(), user.role)`
    6. Hash and store new refresh token
    7. Return authResponse

    **Step 4: Add refresh route**
    ```kotlin
    post("/refresh") {
        conduit {
            val request: RefreshTokenRequest = getModel()
            authService.refresh(request)
        }
    }
    ```

    **Step 5: Add logout to AuthService**
    Add `context(raise: Raise<DomainError>) suspend fun logout(userId: String): Map<String, String>` to AuthService:
    1. Parse userId to UUID (use `UUID.fromString(userId)` -- this should be valid since it comes from JWT)
    2. Revoke all refresh tokens for user: `refreshTokenRepository.revokeByUserId(uuid)`
    3. Return `mapOf("message" to "Logged out successfully")`

    **Step 6: Add logout route (authenticated)**
    ```kotlin
    authenticate {
        post("/logout") {
            conduitAuth { userId ->
                authService.logout(userId)
            }
        }
    }
    ```
    The `authenticate` block uses Ktor's default JWT authentication (configured in SecurityPlugin.kt). The `conduitAuth` wrapper extracts userId from the JWT principal.

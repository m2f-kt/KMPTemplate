Migrate all SDK client-side HTTP calls from hardcoded URL strings to type-safe `client.get(Resource())` / `client.post(Resource())` calls, and update AuthInterceptor to use the typed resource for refresh endpoint detection.

Purpose: Eliminate all hardcoded URL strings from the client SDK, achieving compile-time route safety on the client side. Route changes in ApiRoutes.kt will now cause compiler errors in both server handlers AND client API calls.

Output: Zero hardcoded API URL strings remain in the SDK. AuthInterceptor uses typed resource for refresh detection. Full stack is type-safe.
</objective>

    **AuthApi.kt migration (6 calls):**

    IMPORTANT: Use `import io.ktor.client.plugins.resources.post` (NOT `io.ktor.client.request.post`). The client-resources variant accepts a resource instance parameter.

    Replace each string URL call with the typed resource equivalent:

    - `client.post("/api/auth/register") { ... }` -> `client.post(Auth.Register()) { ... }`
    - `client.post("/api/auth/login") { ... }` -> `client.post(Auth.Login()) { ... }`
    - `client.post("/api/auth/refresh") { ... }` -> `client.post(Auth.Refresh()) { ... }`
    - `client.post("/api/auth/forgot-password") { ... }` -> `client.post(Auth.ForgotPassword()) { ... }`
    - `client.post("/api/auth/reset-password") { ... }` -> `client.post(Auth.ResetPassword()) { ... }`
    - `client.post("/api/auth/logout")` -> `client.post(Auth.Logout())`

    Import changes:
    - Add: `import io.ktor.client.plugins.resources.post` (type-safe client resources post)
    - Add: `import com.m2f.template.models.routes.Auth`
    - Remove: `import io.ktor.client.request.post` (replaced by resources variant)

    The `contentType(ContentType.Application.Json)` and `setBody(request)` calls inside the request builder lambdas remain UNCHANGED. The type-safe `post(Resource()) { ... }` accepts the same builder lambda.

    **UserApi.kt migration (3 calls):**

    - `client.get("/api/users/me")` -> `client.get(Users.Me())`
    - `client.put("/api/users/me") { ... }` -> `client.put(Users.Me()) { ... }`
    - `client.get("/api/users/$id")` -> `client.get(Users.ById(id = id))`

    Import changes:
    - Add: `import io.ktor.client.plugins.resources.get` (type-safe client resources get)
    - Add: `import io.ktor.client.plugins.resources.put` (type-safe client resources put)
    - Add: `import com.m2f.template.models.routes.Users`
    - Remove: `import io.ktor.client.request.get` and `import io.ktor.client.request.put` (replaced by resources variants)

    Key note for `getUserById`: The string interpolation `"/api/users/$id"` is replaced by `Users.ById(id = id)` -- the path parameter is now a typed property on the resource class. This provides compile-time safety: if the parameter name changes, it's a compiler error.

    **AuthInterceptor.kt migration (2 references):**

    1. **Refresh endpoint detection** (line ~37):
       Current: `val isRefreshRequest = request.url.buildString().contains("/auth/refresh")`
       Replace with: `val isRefreshRequest = request.url.encodedPath.endsWith("/api/auth/refresh")`
       Note: The research found that `href()` from ktor-resources might need an Application context. Use a simpler approach: compare against the known path. To keep it tied to the resource definition, define the path check using a constant. The simplest reliable approach is:
       ```kotlin
       val refreshPath = "/api/auth/refresh"  // Matches Auth.Refresh resource path
       val isRefreshRequest = request.url.encodedPath.endsWith(refreshPath)
       ```

       HOWEVER, for maximum type-safety, try using `href(ResourcesFormat(), Auth.Refresh())` first. The `href()` function from `io.ktor.resources` takes a `ResourcesFormat` and a resource instance, returning the URL path string. This is available in shared/client code because `ktor-resources` is multiplatform.

       If `href(ResourcesFormat(), Auth.Refresh())` compiles and works:
       ```kotlin
       import io.ktor.resources.href
       import io.ktor.resources.ResourcesFormat
       import com.m2f.template.models.routes.Auth

       // At class level or in install():
       private val refreshPath = href(ResourcesFormat(), Auth.Refresh())

       // In the intercept lambda:
       val isRefreshRequest = request.url.encodedPath == refreshPath
       ```

       If `href()` doesn't work in client context (compile error), fall back to:
       ```kotlin
       val isRefreshRequest = request.url.encodedPath.endsWith("/api/auth/refresh")
       ```
       Either way, document the connection to Auth.Refresh in a comment.

    2. **Refresh request call** (line ~64):
       Current: `client.post("/api/auth/refresh") { ... }`
       Replace with: `client.post(Auth.Refresh()) { ... }`

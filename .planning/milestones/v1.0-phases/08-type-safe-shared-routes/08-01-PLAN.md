    1. Create `core/models/src/commonMain/kotlin/com/m2f/template/models/routes/ApiRoutes.kt` with all @Resource-annotated route classes. Use this exact structure from the research:

    ```kotlin
    package com.m2f.template.models.routes

    import io.ktor.resources.Resource
    import kotlinx.serialization.Serializable

    @Serializable
    @Resource("/api/auth")
    class Auth {
        @Serializable @Resource("register")
        class Register(val parent: Auth = Auth())

        @Serializable @Resource("login")
        class Login(val parent: Auth = Auth())

        @Serializable @Resource("refresh")
        class Refresh(val parent: Auth = Auth())

        @Serializable @Resource("logout")
        class Logout(val parent: Auth = Auth())

        @Serializable @Resource("forgot-password")
        class ForgotPassword(val parent: Auth = Auth())

        @Serializable @Resource("reset-password")
        class ResetPassword(val parent: Auth = Auth())
    }

    @Serializable
    @Resource("/api/users")
    class Users {
        @Serializable @Resource("me")
        class Me(val parent: Users = Users())

        @Serializable @Resource("{id}")
        class ById(val parent: Users = Users(), val id: String)
    }

    @Serializable
    @Resource("/api/ai")
    class Ai {
        @Serializable @Resource("assistant")
        class Assistant(val parent: Ai = Ai())

        @Serializable @Resource("chat")
        class Chat(val parent: Ai = Ai()) {
            companion object {
                /** WebSocket path -- type-safe routing not supported for WebSockets (KTOR-4369) */
                const val WS_PATH = "/api/ai/chat/ws"
            }
        }
    }
    ```

    Key design notes:
    - Regular classes (not data classes) for resources without properties -- they need no-arg constructors.
    - `Users.ById` has `val id: String` matching the `{id}` path parameter exactly (name must match).
    - `Ai.Chat.WS_PATH` companion constant because WebSocket doesn't support type-safe routing (KTOR-4369).
    - Each nested class has `val parent: ParentType = ParentType()` for Ktor's nested resource resolution.

    2. In `server/src/main/kotlin/com/m2f/template/Application.kt`, add `install(Resources)` in `Application.module()`:
       - Add import: `import io.ktor.server.resources.Resources`
       - Add `install(Resources)` AFTER `install(Koin)` and BEFORE `install(ContentNegotiation)`.
       - This is the SERVER Resources plugin from `io.ktor.server.resources`.

    3. In `core/sdk/src/commonMain/kotlin/com/m2f/template/sdk/ApiClient.kt`, add `install(Resources)` in the HttpClient builder:
       - Add import: `import io.ktor.client.plugins.resources.Resources`
       - Add `install(Resources)` inside the `HttpClient(platformEngine()) { ... }` block, AFTER `install(ContentNegotiation)`.
       - This is the CLIENT Resources plugin from `io.ktor.client.plugins.resources`.
       - IMPORTANT: Different import path than server! `io.ktor.client.plugins.resources.Resources` not `io.ktor.server.resources.Resources`.

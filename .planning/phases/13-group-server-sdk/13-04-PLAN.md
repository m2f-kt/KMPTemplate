**Test organization:**

```kotlin
class GroupRoutesTest {
    companion object {
        @BeforeAll @JvmStatic
        fun setup() { TestDatabase.start() }

        @AfterAll @JvmStatic
        fun teardown() { TestDatabase.stop() }
    }

    // Group CRUD tests
    @Test fun `admin can create a group`()
    @Test fun `create group returns 403 for regular user`()
    @Test fun `create group with duplicate slug returns error`()
    @Test fun `admin can read their own group`()
    @Test fun `admin can update their group`()
    @Test fun `owner can delete their group`()
    @Test fun `power admin can list all groups`()
    @Test fun `non-power-admin cannot list all groups`()

    // Member management tests
    @Test fun `admin can add existing user to group`()
    @Test fun `admin can remove member from group`()
    @Test fun `cannot remove group owner`()
    @Test fun `admin can register new user into group`()
    @Test fun `admin can list group members with pagination`()
    @Test fun `regular member cannot list group members`()

    // RBAC tests
    @Test fun `group member cannot update group`()
    @Test fun `group member cannot add members`()
    @Test fun `group admin can manage members but not delete group`()

    // Cross-group isolation tests
    @Test fun `user in group A cannot access group B - returns 403`()
    @Test fun `power admin can access any group`()
}
```

**Test implementation pattern for each test:**
1. Use `testApp { }` helper to set up test application
2. Create test users via `createTestUser()` with appropriate roles
3. Generate JWT tokens via `createTestToken()` for different users
4. Create groups and memberships as needed via API calls or direct DB inserts
5. Make HTTP requests with appropriate auth headers
6. Assert response status codes and body content using kotest assertions (`shouldBe`, `shouldNotBeNull`, etc.)

**Key test scenarios:**

**Auth flow test:** Register admin user -> login -> create group -> verify group persists

**Cross-group isolation test:**
1. Create two groups (Group A, Group B)
2. Create User1 (member of Group A only) and User2 (member of Group B only)
3. User1 tries to GET /api/groups/{groupB_id} -> expects 403 Forbidden
4. User1 tries to GET /api/groups/{groupA_id} -> expects 200 OK

**RBAC test:**
1. Create group with Owner, Admin, and Member users
2. Member tries POST /api/groups/{id}/update -> expects 403
3. Admin tries POST /api/groups/{id}/update -> expects 200
4. Member tries POST /api/groups/{id}/members/add -> expects 403
5. Admin tries POST /api/groups/{id}/members/add -> expects 200

**Pagination test:**
1. Create group with 25 members
2. GET /api/groups/{id}/members?limit=10 -> expect 10 items, hasMore=true, cursor present
3. GET /api/groups/{id}/members?limit=10&cursor={cursor} -> expect 10 items, hasMore=true
4. GET /api/groups/{id}/members?limit=10&cursor={cursor2} -> expect 5 items, hasMore=false

Use kotest assertions throughout:
```kotlin
response.status shouldBe HttpStatusCode.Created
body.name shouldBe "Test Group"
body.slug shouldBe "test-group"
errorBody.code shouldBe "GROUP_FORBIDDEN"
```

   ```kotlin
   /** User wants to request a new invitation (current one is expired). */
   data object RequestNewInvitation : InviteAcceptIntent
   ```

2. **InviteAcceptEvent.kt** -- Add a new event:
   ```kotlin
   /** Navigate away after requesting a new invitation (expired token). */
   data object RequestedNewInvitation : InviteAcceptEvent
   ```

3. **InviteAcceptViewModel.kt** -- In the `take()` `when` block, add handler for the new intent:
   ```kotlin
   is InviteAcceptIntent.RequestNewInvitation -> {
       sendEvent(InviteAcceptEvent.RequestedNewInvitation)
   }
   ```
   This is intentionally simple -- the template demonstrates the MVI pattern. A real app would call an API to notify the admin.

4. **String resources (EN)** -- In `values/strings.xml`, after the `invite_auth_prompt` line (line 152), add before `</resources>`:
   ```xml
   <!-- Invite Expired Action -->
   <string name="invite_request_new">$ request_new_invite()</string>
   <string name="invite_expired_hint">// contact the group administrator to receive a new invitation</string>
   ```

5. **String resources (ES)** -- In `values-es/strings.xml`, at the same position, add:
   ```xml
   <!-- Invite Expired Action -->
   <string name="invite_request_new">$ solicitar_nueva_invitacion()</string>
   <string name="invite_expired_hint">// contacta al administrador del grupo para recibir una nueva invitacion</string>
   ```

6. **InviteAcceptScreen.kt** -- Add a new callback parameter and the button display:
   - Add `onRequestNewInvitation: () -> Unit` parameter to `InviteAcceptScreen` composable (after `onGoToRegister`).
   - Add the import for the two new string resources: `import template.app.auth.generated.resources.invite_request_new` and `import template.app.auth.generated.resources.invite_expired_hint`.
   - Inside the expired block (after the `TerminalAlert` at lines 139-143), add:
     ```kotlin
     // Expired alert
     if (state.isExpired) {
         TerminalAlert(
             message = stringResource(Res.string.invite_expired),
             variant = AlertVariant.Warning,
         )
         Spacer(modifier = Modifier.height(8.dp))
         TerminalText(
             text = stringResource(Res.string.invite_expired_hint),
             style = typography.sm,
             color = colors.textDim,
         )
         Spacer(modifier = Modifier.height(8.dp))
         TerminalButton(
             text = stringResource(Res.string.invite_request_new),
             onClick = onRequestNewInvitation,
             modifier = Modifier.fillMaxWidth(),
             variant = ButtonVariant.Secondary,
         )
     }
     ```
     This places the hint and button directly below the expired alert, inside the card but outside the `!state.isExpired` action button guard. The button uses `ButtonVariant.Secondary` consistent with the register button styling convention.

7. **AppNavHost.kt** -- In the `composable<InviteAcceptRoute>` block:
   - Add `onRequestNewInvitation` callback to the `InviteAcceptScreen` call:
     ```kotlin
     onRequestNewInvitation = { viewModel.take(InviteAcceptIntent.RequestNewInvitation) },
     ```
   - In the `viewModel.event.collect` `when` block, add handler:
     ```kotlin
     is InviteAcceptEvent.RequestedNewInvitation -> {
         navController.navigate(LoginRoute()) {
             popUpTo<InviteAcceptRoute> { inclusive = true }
         }
     }
     ```
     This navigates to the login screen as a sensible default for a template -- the user can log in and contact the admin, or register a new account. The popUpTo clears the expired invite screen from the back stack.

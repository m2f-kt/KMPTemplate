## Decisions Made
- **Callback pattern over direct dependency:** The plan suggested injecting `InvitationService` into `AuthService` via `AuthModule`, but `server/auth` does not depend on `server/groups` (and `server/groups` depends on `server/auth`), creating a circular dependency. Solution: `AuthService` takes an optional `onRegistered` callback, wired at the composition root (`ServerModule.kt`).
- **Koin dual registration:** `AuthService` registered in `authModule` (default, no callback) so existing tests and `GroupService` dependency resolution work. `serverModule` overrides with the enriched version. Koin 3.1.0+ allows last-definition-wins by default.
- **Regular function parameter for Raise:** Using `context(Raise<DomainError>)` in a function type (`suspend (Raise<DomainError>, String, RegisterRequest) -> Unit`) caused `SuspendFunction4` vs `SuspendFunction3` arity mismatches. Solution: pass `raise: Raise<DomainError>` as a regular parameter and use `with(raise) { ... }` at the call site.

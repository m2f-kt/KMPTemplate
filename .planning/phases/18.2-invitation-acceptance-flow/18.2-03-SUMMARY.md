# Phase 18.2 Plan 03: Login/Register Invitation Token Integration Summary

Post-login invitation acceptance via sdk.acceptInvitation() and registration with invitationToken in RegisterRequest, completing the end-to-end invitation flow from email link through authentication to group membership.

## What Was Done

### Task 1: Add invitationToken to Routes and Login MVI
- Changed `LoginRoute` and `RegisterRoute` from `data object` to `data class` with optional `invitationToken: String? = null`
- Added `invitationToken` and `isAcceptingInvitation` fields to `LoginModel`
- Added `SetInvitationToken` intent, `SetInvitationToken`/`SetAcceptingInvitation` mutations
- Added `NavigateToGroup(groupId)` event to `LoginEvent`
- Implemented `handlePostLogin()` in `LoginViewModel`: when invitation token is present, calls `sdk.acceptInvitation()` after successful login; navigates to AdminPanelRoute on success, falls back to dashboard on failure

### Task 2: Add invitationToken to Register MVI
- Added `invitationToken: String? = null` to `RegisterModel`
- Added `SetInvitationToken` intent and mutation to Register MVI layer
- Added `NavigateToGroup(groupId)` event to `RegisterEvent`
- Modified `RegisterViewModel.handleRegister()` to include `invitationToken = current.invitationToken` in `RegisterRequest`
- Server handles invitation acceptance automatically during registration when token is present

### Task 3: Wire AppNavHost for Complete Invitation Flow
- Updated `AppNavHost` startDestination to `LoginRoute()` (constructor call)
- `composable<LoginRoute>`: extracts route, passes invitationToken to ViewModel via LaunchedEffect, handles `NavigateToGroup` event
- `composable<RegisterRoute>`: extracts route, passes invitationToken to ViewModel via LaunchedEffect, handles `NavigateToGroup` event
- `composable<InviteAcceptRoute>`: navigates to `LoginRoute(invitationToken = event.token)` and `RegisterRoute(invitationToken = event.token)` with popUpTo
- Updated all `navigate(LoginRoute)` → `navigate(LoginRoute())` and `navigate(RegisterRoute)` → `navigate(RegisterRoute())` across AppNavHost and App.kt

## Complete Flow

1. User receives invitation email with link → navigates to `/invite/accept?token=...`
2. App extracts token → navigates to `InviteAcceptRoute(token)`
3. `InviteAcceptScreen` shows invitation details (group name, inviter)
4. User clicks "Login" → `InviteAcceptEvent.NavigateToLogin(token)` → `LoginRoute(invitationToken = token)`
5. User logs in → `LoginViewModel.handlePostLogin()` → `sdk.acceptInvitation(token)` → `NavigateToGroup(groupId)` → `AdminPanelRoute`
6. OR User clicks "Register" → `InviteAcceptEvent.NavigateToRegister(token)` → `RegisterRoute(invitationToken = token)`
7. User registers → `RegisterRequest(invitationToken = token)` → server auto-links → `NavigateToDashboard` (MVP)

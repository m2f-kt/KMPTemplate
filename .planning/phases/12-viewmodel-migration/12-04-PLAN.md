<objective>
Migrate ProfileViewModel to the MVI pattern, following the LoginViewModel reference from Plan 01.

Purpose: ProfileViewModel has two key distinctions: (1) it takes two separate API dependencies (UserApi + AuthApi) that must be consolidated to a single Sdk, (2) logoutTriggered becomes a NavigateToLogin Event while saveSuccess stays as Model state. It also has init-time profile loading via the intent pipeline.
Output: Fully MVI-migrated ProfileViewModel with tests and updated AppNavHost ProfileRoute block.
</objective>

    **Rewrite ProfileViewModel.kt:**
    - Change constructor from `userApi: UserApi, authApi: AuthApi` to single `sdk: Sdk`
    - Extend `MviViewModel<ProfileIntent, ProfileModel, ProfileMutation, ProfileEvent>(initialState = ProfileModel())`
    - `init { take(ProfileIntent.LoadProfile) }` -- load profile via intent pipeline
    - `take(intent)` in `viewModelScope.launch`:
      - `LoadProfile` -> `sendMutation(SetLoading(true))`, call `sdk.getProfile()`.fold -- ifLeft: `sendMutation(SetServerError(error.message))`, `sendMutation(SetLoading(false))`; ifRight: `sendMutation(SetProfile(user.id, user.email, user.name, user.tier))`
      - `StartEditing` -> `sendMutation(StartEdit)`
      - `CancelEditing` -> `sendMutation(CancelEdit)`
      - `EditNameChanged` -> `sendMutation(SetEditName(intent.name))`
      - `EditEmailChanged` -> `sendMutation(SetEditEmail(intent.email))`
      - `SaveProfileClicked` -> validate edit fields (same logic as current). If errors: `sendMutation(SetFieldErrors(errorMap))`. If valid: `sendMutation(SetLoading(true))`, call `sdk.updateProfile(...)`.fold -- ifLeft: `sendMutation(SetServerError(error.message))`, `sendMutation(SetLoading(false))`; ifRight: update model with new data + `sendMutation(SetSaveSuccess)`
      - `LogoutClicked` -> call `sdk.logout()`, then `sendEvent(ProfileEvent.NavigateToLogin)` regardless of result (always navigate to login on logout attempt)
    - `reduce(model, mutation)` -- pure:
      - SetProfile: `model.copy(userId = ..., email = ..., name = ..., tier = ..., isLoading = false)`
      - SetLoading: `model.copy(isLoading = mutation.loading)`
      - StartEdit: `model.copy(isEditing = true, editName = model.name, editEmail = model.email, saveSuccess = false, fieldErrors = emptyMap())`
      - CancelEdit: `model.copy(isEditing = false, editName = "", editEmail = "", fieldErrors = emptyMap())`
      - SetEditName: `model.copy(editName = mutation.name, fieldErrors = model.fieldErrors - "name")`
      - SetEditEmail: `model.copy(editEmail = mutation.email, fieldErrors = model.fieldErrors - "email")`
      - SetFieldErrors: `model.copy(fieldErrors = mutation.errors)`
      - SetServerError: `model.copy(serverError = mutation.error, isLoading = false)`
      - SetSaveSuccess: `model.copy(isEditing = false, saveSuccess = true, isLoading = false)` -- also update name/email from edit fields

    **Update ProfileScreen.kt:** Change `state: ProfileState` to `state: ProfileModel`. Update the import. Keep all callback parameters the same.

    **Update AppNavHost.kt ProfileRoute block:**

      - `onEditNameChange = { viewModel.take(ProfileIntent.EditNameChanged(it)) }`

      - `onSaveProfile = { viewModel.take(ProfileIntent.SaveProfileClicked) }`

      ```kotlin
                          popUpTo(0) { inclusive = true }
                      }
                  }
              }
          }
      }
      ```

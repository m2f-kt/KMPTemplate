Rewrite ProfileViewModelTest.kt to follow the exact same pattern as DashboardViewModelTest and LoginViewModelTest. Specifically:

1. **Class declaration:** Replace `class ProfileViewModelTest {` with `class ProfileViewModelTest : ViewModelTest() {`

2. **Remove manual dispatcher boilerplate:** Delete the `testDispatcher` field, `@BeforeTest setUp()`, and `@AfterTest tearDown()` methods entirely. ViewModelTest base class handles this.

3. **Remove all raw coroutine imports:** Delete imports for `Dispatchers`, `ExperimentalCoroutinesApi`, `UnconfinedTestDispatcher`, `advanceUntilIdle`, `resetMain`, `runTest`, `setMain`, `turbineScope`. Delete the `@OptIn(ExperimentalCoroutinesApi::class)` annotation.

4. **Add DSL imports:** Add `import com.m2f.template.core.testing.ViewModelTest`, `import com.m2f.template.core.testing.test`.

5. **Rewrite all 4 tests using viewModel.test {}:**

   **Test: `load profile populates model with user data`**
   - ProfileViewModel has `init { take(ProfileIntent.LoadProfile) }` -- same pattern as DashboardViewModel
   - Follow DashboardViewModelTest pattern: explicitly fire `intent(ProfileIntent.LoadProfile)` inside the DSL
   - With sync fake SDK, SetLoading(true) + SetProfile conflate via StateFlow, so expect only the final loaded state
   - DSL block:
     ```
     intent(ProfileIntent.LoadProfile)
     model(ProfileModel(userId = "u1", email = "a@b.com", name = "Test User", tier = UserTier.Free, isLoading = false))
     ```

   **Test: `save profile shows success in model`**
   - Start with LoadProfile intent to get loaded state (same init-dispatch pattern)
   - Then dispatch intents sequentially: StartEditing, EditNameChanged, EditEmailChanged, SaveProfileClicked
   - Each intent produces a model assertion
   - StateFlow conflation applies to SaveProfileClicked: SetLoading + SetProfile + SetSaveSuccess conflate to final state
   - DSL block:
     ```
     intent(ProfileIntent.LoadProfile)
     model(ProfileModel(userId = "u1", email = "a@b.com", name = "Test User", tier = UserTier.Free, isLoading = false))
     intent(ProfileIntent.StartEditing)
     model(ProfileModel(userId = "u1", email = "a@b.com", name = "Test User", tier = UserTier.Free, isLoading = false, isEditing = true, editName = "Test User", editEmail = "a@b.com"))
     intent(ProfileIntent.EditNameChanged("New Name"))
     model(ProfileModel(userId = "u1", email = "a@b.com", name = "Test User", tier = UserTier.Free, isLoading = false, isEditing = true, editName = "New Name", editEmail = "a@b.com"))
     intent(ProfileIntent.EditEmailChanged("new@b.com"))
     model(ProfileModel(userId = "u1", email = "a@b.com", name = "Test User", tier = UserTier.Free, isLoading = false, isEditing = true, editName = "New Name", editEmail = "new@b.com"))
     intent(ProfileIntent.SaveProfileClicked)
     model(ProfileModel(userId = "u1", email = "new@b.com", name = "New Name", tier = UserTier.Free, isLoading = false, isEditing = false, editName = "New Name", editEmail = "new@b.com", saveSuccess = true))
     ```

   **Test: `load profile failure shows server error`**
   - Same init-dispatch pattern with error SDK
   - StateFlow conflation: SetLoading(true) + SetServerError + SetLoading(false) conflate to final error state
   - DSL block:
     ```
     intent(ProfileIntent.LoadProfile)
     model(ProfileModel(isLoading = false, serverError = "Network error"))
     ```

   **Test: `logout emits NavigateToLogin event`**
   - Start with LoadProfile to reach loaded state
   - Then dispatch LogoutClicked
   - Assert the NavigateToLogin event
   - DSL block:
     ```
     intent(ProfileIntent.LoadProfile)
     model(ProfileModel(userId = "u1", email = "a@b.com", name = "Test User", tier = UserTier.Free, isLoading = false))
     intent(ProfileIntent.LogoutClicked)
     event(ProfileEvent.NavigateToLogin)
     ```

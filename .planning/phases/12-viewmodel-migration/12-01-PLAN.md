  <action>
    **Build config:** Add `implementation(projects.core.mvi)` to `app/auth/build.gradle.kts` commonMain.dependencies block.

    **Create 4 type files** in `app/auth/src/commonMain/kotlin/com/m2f/template/app/auth/`:

    1. **LoginIntent.kt** -- sealed interface with:
       - `data class EmailChanged(val email: String)`
       - `data class PasswordChanged(val password: String)`
       - `data class RememberMeChanged(val checked: Boolean)`
       - `data object SubmitLoginClicked`

    2. **LoginModel.kt** -- data class replacing LoginState, with the SAME fields EXCEPT `loginSuccess` which is removed (becomes an Event):
       ```kotlin
       data class LoginModel(
           val email: String = "",
           val password: String = "",
           val rememberMe: Boolean = false,
           val isLoading: Boolean = false,
           val emailError: String? = null,
           val passwordError: String? = null,
           val serverError: String? = null,
       )
       ```

    3. **LoginMutation.kt** -- sealed interface with:
       - `data class SetEmail(val email: String)`
       - `data class SetPassword(val password: String)`
       - `data class SetRememberMe(val checked: Boolean)`
       - `data class SetLoading(val loading: Boolean)`
       - `data class SetValidationErrors(val emailError: String?, val passwordError: String?)`
       - `data class SetServerError(val error: String?)`

    4. **LoginEvent.kt** -- sealed interface with:
       - `data object NavigateToDashboard`

    **Rewrite LoginViewModel.kt** to extend `MviViewModel<LoginIntent, LoginModel, LoginMutation, LoginEvent>(initialState = LoginModel())`:
    - Constructor stays `(private val sdk: Sdk)` -- already correct from Phase 11.1
    - `take(intent)` dispatches in `viewModelScope.launch`:
      - `EmailChanged` -> `sendMutation(SetEmail(intent.email))`
      - `PasswordChanged` -> `sendMutation(SetPassword(intent.password))`
      - `RememberMeChanged` -> `sendMutation(SetRememberMe(intent.checked))`
      - `SubmitLoginClicked` -> runs validation. If errors: `sendMutation(SetValidationErrors(...))` and return. If valid: `sendMutation(SetLoading(true))`, call `sdk.login(...)`.fold -- ifLeft: `sendMutation(SetServerError(error.message))`, `sendMutation(SetLoading(false))`; ifRight: `sendEvent(LoginEvent.NavigateToDashboard)` (NO model update after successful login -- per user decision, login success fires NavigateToDashboard immediately with no success state)
    - `reduce(model, mutation)` is a pure function mapping each Mutation to model.copy(...)
      - SetEmail: `model.copy(email = mutation.email, emailError = null)` (clear error on change)
      - SetPassword: `model.copy(password = mutation.password, passwordError = null)`
      - SetRememberMe: `model.copy(rememberMe = mutation.checked)`
      - SetLoading: `model.copy(isLoading = mutation.loading, serverError = null)` (clear server error on new attempt)
      - SetValidationErrors: `model.copy(emailError = mutation.emailError, passwordError = mutation.passwordError)`
      - SetServerError: `model.copy(serverError = mutation.error, isLoading = false)`

    **Update LoginScreen.kt:** Change the `state: LoginState` parameter to `state: LoginModel`. Update the import from `LoginState` to `LoginModel`. Keep all callback parameters exactly the same. The Screen composable remains ViewModel-agnostic. Per "migration purity" decision -- same behavior, just new type name.

    **Update AppNavHost.kt LoginRoute block:**
    - Change `viewModel.state` to `viewModel.model`
    - Change `val state by viewModel.model.collectAsStateWithLifecycle()` -- rename local from `state` to `model` (or keep as `state` for minimal churn in LoginScreen usage -- keep as `state` since the Screen still uses that name in its parameter)
    - Replace individual method call lambdas with intent dispatch: `onEmailChange = { viewModel.take(LoginIntent.EmailChanged(it)) }`, `onPasswordChange = { viewModel.take(LoginIntent.PasswordChanged(it)) }`, `onRememberMeChange = { viewModel.take(LoginIntent.RememberMeChanged(it)) }`, `onLoginClick = { viewModel.take(LoginIntent.SubmitLoginClicked) }`
    - REMOVE the `LaunchedEffect(state.loginSuccess)` block entirely
    - ADD a new `LaunchedEffect(Unit)` block AFTER the LoginScreen call:
      ```kotlin
      LaunchedEffect(Unit) {
          viewModel.event.collect { event ->
              when (event) {
                  is LoginEvent.NavigateToDashboard -> {
                      navController.navigate(DashboardRoute) {
                          popUpTo<LoginRoute> { inclusive = true }
                      }
                  }
              }
          }
      }
      ```

    **Important:** The LoginScreen composable parameter name stays as `state: LoginModel` (not renamed to `model`) to minimize downstream changes in the Screen composable body that references `state.email`, `state.password`, etc. The AppNavHost passes the collected model as `state = state`.
  </action>

  </verify>

    Test 1: `successful login emits NavigateToDashboard event`
    ```kotlin
    val sdk = fakeSdk {
        auth { login { _, _ -> Either.Right(AuthResponse(accessToken = "tok", refreshToken = "ref", expiresIn = 3600)) } }
    }
    val viewModel = LoginViewModel(sdk)
    viewModel.test {
        intent(LoginIntent.EmailChanged("user@test.com"))
        model(LoginModel(email = "user@test.com"))
        intent(LoginIntent.PasswordChanged("password123"))
        model(LoginModel(email = "user@test.com", password = "password123"))
        intent(LoginIntent.SubmitLoginClicked)
        model(LoginModel(email = "user@test.com", password = "password123", isLoading = true))
        event(LoginEvent.NavigateToDashboard)
    }
    ```
    Note: After `NavigateToDashboard` event, no further model emission is expected (per user decision: login success fires event immediately, no state update).

    Test 2: `blank email shows validation error`
    ```kotlin
    val sdk = fakeSdk()
    val viewModel = LoginViewModel(sdk)
    viewModel.test {
        intent(LoginIntent.SubmitLoginClicked)
        model(LoginModel(emailError = "Email must not be blank", passwordError = "Password must not be blank"))
    }
    ```

    Test 3: `server error shows error in model`
    ```kotlin
    val sdk = fakeSdk {
        auth { login { _, _ -> Either.Left(AppError.Auth.InvalidCredentials()) } }
    }
    val viewModel = LoginViewModel(sdk)
    viewModel.test {
        intent(LoginIntent.EmailChanged("user@test.com"))
        model(LoginModel(email = "user@test.com"))
        intent(LoginIntent.PasswordChanged("password123"))
        model(LoginModel(email = "user@test.com", password = "password123"))

        model(LoginModel(email = "user@test.com", password = "password123", serverError = "Email or password is incorrect"))
    }
    ```

  <action>
    **Read the current ForgotPasswordViewModel.kt and ForgotPasswordState.kt** to understand existing behavior.

    **Create 4 type files** in `app/auth/src/commonMain/kotlin/com/m2f/template/app/auth/`:

    1. **ForgotPasswordIntent.kt** -- sealed interface with:
       - `data class EmailChanged(val email: String)`
       - `data object SubmitForgotPasswordClicked`

    2. **ForgotPasswordModel.kt** -- data class replacing ForgotPasswordState. **IMPORTANT: `emailSent` STAYS as a Model field** (per Pitfall 6 in research -- user stays on screen seeing success message):
       ```kotlin
       data class ForgotPasswordModel(
           val email: String = "",
           val isLoading: Boolean = false,
           val emailSent: Boolean = false,
           val emailError: String? = null,
           val serverError: String? = null,
       )
       ```

    3. **ForgotPasswordMutation.kt** -- sealed interface with:
       - `data class SetEmail(val email: String)`
       - `data class SetLoading(val loading: Boolean)`
       - `data object SetEmailSent`
       - `data class SetEmailError(val error: String?)`
       - `data class SetServerError(val error: String?)`

    4. **ForgotPasswordEvent.kt** -- **empty sealed interface**. ForgotPassword has no ViewModel-driven navigation. "Back to login" is handled by the Composable callback directly:
       ```kotlin
       sealed interface ForgotPasswordEvent
       ```

    **Rewrite ForgotPasswordViewModel.kt:**
    - Change constructor from `authApi: AuthApi` to `sdk: Sdk`
    - Extend `MviViewModel<ForgotPasswordIntent, ForgotPasswordModel, ForgotPasswordMutation, ForgotPasswordEvent>(initialState = ForgotPasswordModel())`
    - `take(intent)` in `viewModelScope.launch`:
      - `EmailChanged` -> `sendMutation(SetEmail(intent.email))`
      - `SubmitForgotPasswordClicked` -> validate email (same logic as current). If error: `sendMutation(SetEmailError(...))`. If valid: `sendMutation(SetLoading(true))`, call `sdk.forgotPassword(...)`.fold -- ifLeft: `sendMutation(SetServerError(error.message))`, `sendMutation(SetLoading(false))`; ifRight: `sendMutation(SetEmailSent)` (Model state, NOT event)
    - `reduce(model, mutation)` -- pure:
      - SetEmail: `model.copy(email = mutation.email, emailError = null)`
      - SetLoading: `model.copy(isLoading = mutation.loading, serverError = null)`
      - SetEmailSent: `model.copy(isLoading = false, emailSent = true)`
      - SetEmailError: `model.copy(emailError = mutation.error)`
      - SetServerError: `model.copy(serverError = mutation.error, isLoading = false)`

    **Update ForgotPasswordScreen.kt:** Change `state: ForgotPasswordState` to `state: ForgotPasswordModel`. Update the import. Keep callbacks the same.

    **Update AppNavHost.kt ForgotPasswordRoute block:**
    - `viewModel.state` -> `viewModel.model`
    - Replace method references: `onEmailChange = { viewModel.take(ForgotPasswordIntent.EmailChanged(it)) }`, `onSubmit = { viewModel.take(ForgotPasswordIntent.SubmitForgotPasswordClicked) }`
    - No LaunchedEffect for events needed (ForgotPasswordEvent is empty, no ViewModel-driven navigation)
    - `onBackToLogin` stays as `{ navController.popBackStack() }` -- Composable-level, not ViewModel
  </action>

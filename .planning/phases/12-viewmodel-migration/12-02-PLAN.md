  </files>
  <action>
    **Read the current RegisterViewModel.kt and RegisterState.kt** to understand existing validation logic (Arrow zipOrAccumulate accumulated validation pattern). The validation logic MUST be preserved exactly -- same behavior, new MVI wrapping.

    **Create 4 type files** in `app/auth/src/commonMain/kotlin/com/m2f/template/app/auth/`:

    1. **RegisterIntent.kt** -- sealed interface with:
       - `data class FirstNameChanged(val firstName: String)`
       - `data class LastNameChanged(val lastName: String)`
       - `data class EmailChanged(val email: String)`
       - `data class PasswordChanged(val password: String)`
       - `data class ConfirmPasswordChanged(val confirmPassword: String)`
       - `data class TermsAcceptedChanged(val accepted: Boolean)`
       - `data object SubmitRegisterClicked`

    2. **RegisterModel.kt** -- data class replacing RegisterState. Copy ALL fields from RegisterState EXCEPT `registerSuccess` (becomes an Event). Keep fieldErrors as `Map<String, String>`:
       ```kotlin
       data class RegisterModel(
           val firstName: String = "",
           val lastName: String = "",
           val email: String = "",
           val password: String = "",
           val confirmPassword: String = "",
           val termsAccepted: Boolean = false,
           val isLoading: Boolean = false,
           val fieldErrors: Map<String, String> = emptyMap(),
           val serverError: String? = null,
       )
       ```

    3. **RegisterMutation.kt** -- sealed interface with mutations for each field change plus loading/errors:
       - `data class SetFirstName(val firstName: String)`
       - `data class SetLastName(val lastName: String)`
       - `data class SetEmail(val email: String)`
       - `data class SetPassword(val password: String)`
       - `data class SetConfirmPassword(val confirmPassword: String)`
       - `data class SetTermsAccepted(val accepted: Boolean)`
       - `data class SetLoading(val loading: Boolean)`
       - `data class SetFieldErrors(val errors: Map<String, String>)`
       - `data class SetServerError(val error: String?)`

    4. **RegisterEvent.kt** -- sealed interface with:
       - `data object NavigateToDashboard`

    **Rewrite RegisterViewModel.kt:**
    - Change constructor from `authApi: AuthApi` to `sdk: Sdk` (per Phase 11.1 decision)
    - Extend `MviViewModel<RegisterIntent, RegisterModel, RegisterMutation, RegisterEvent>(initialState = RegisterModel())`
    - `take(intent)` in `viewModelScope.launch`:
      - Field change intents -> sendMutation for the corresponding field (each clears its own error from fieldErrors)
      - `SubmitRegisterClicked` -> preserve the EXACT validation logic from existing RegisterViewModel (Arrow `either { zipOrAccumulate }` pattern). If errors: `sendMutation(SetFieldErrors(errorMap))`. If valid: `sendMutation(SetLoading(true))`, call `sdk.register(...)`.fold -- ifLeft: `sendMutation(SetServerError(error.message))`, `sendMutation(SetLoading(false))`; ifRight: `sendEvent(RegisterEvent.NavigateToDashboard)` (NO model update after success)
    - `reduce(model, mutation)` -- pure function:
      - SetFirstName: `model.copy(firstName = mutation.firstName, fieldErrors = model.fieldErrors - "firstName")`
      - SetLastName: similar pattern clearing "lastName" error
      - SetEmail: clear "email" error
      - SetPassword: clear "password" error
      - SetConfirmPassword: clear "confirmPassword" error
      - SetTermsAccepted: clear "termsAccepted" error
      - SetLoading: `model.copy(isLoading = mutation.loading, serverError = null)`
      - SetFieldErrors: `model.copy(fieldErrors = mutation.errors)`
      - SetServerError: `model.copy(serverError = mutation.error, isLoading = false)`
    - API call: change `authApi.register(...)` to `sdk.register(...)` -- Sdk delegates to AuthApi

    **Update RegisterScreen.kt:** Change `state: RegisterState` to `state: RegisterModel`. Update the import. Keep all callback parameters the same.

    **Update AppNavHost.kt RegisterRoute block:**
    - `viewModel.state` -> `viewModel.model`
    - Replace method references with intent dispatch lambdas:
      - `onFirstNameChange = { viewModel.take(RegisterIntent.FirstNameChanged(it)) }`
      - `onLastNameChange = { viewModel.take(RegisterIntent.LastNameChanged(it)) }`
      - `onEmailChange = { viewModel.take(RegisterIntent.EmailChanged(it)) }`
      - `onPasswordChange = { viewModel.take(RegisterIntent.PasswordChanged(it)) }`
      - `onConfirmPasswordChange = { viewModel.take(RegisterIntent.ConfirmPasswordChanged(it)) }`
      - `onTermsAcceptedChange = { viewModel.take(RegisterIntent.TermsAcceptedChanged(it)) }`
      - `onRegisterClick = { viewModel.take(RegisterIntent.SubmitRegisterClicked) }`
    - REMOVE `LaunchedEffect(state.registerSuccess)` block
    - ADD `LaunchedEffect(Unit)` for event collection:
      ```kotlin
      LaunchedEffect(Unit) {
          viewModel.event.collect { event ->
              when (event) {
                  is RegisterEvent.NavigateToDashboard -> {
                      navController.navigate(DashboardRoute) {
                          popUpTo<LoginRoute> { inclusive = true }
                      }
                  }
              }
          }
      }
      ```
  </action>

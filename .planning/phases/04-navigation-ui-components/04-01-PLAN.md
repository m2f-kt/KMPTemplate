  <action>
Create the navigation route definitions and NavHost, then wire them into the existing App.kt.

**Routes.kt** (`com.m2f.template.navigation`):
Define @Serializable route objects:
- `@Serializable data object LoginRoute` -- auth entry screen
- `@Serializable data object RegisterRoute` -- signup screen
- `@Serializable data object DashboardRoute` -- main app screen after auth
- `@Serializable data object ProfileRoute` -- user profile

Use `data object` for routes with no parameters. All routes use `kotlinx.serialization.Serializable`.

**AppNavHost.kt** (`com.m2f.template.navigation`):
Create a `@Composable fun AppNavHost()` that:
1. Creates `val navController = rememberNavController()`
2. Creates a `NavHost` with `startDestination = LoginRoute`
3. Registers placeholder composable screens for each route:
   - `composable<LoginRoute>` -- placeholder Box with "Login Screen" text, a button navigating to RegisterRoute, and a button navigating to DashboardRoute (simulating login success)
   - `composable<RegisterRoute>` -- placeholder Box with "Register Screen" text and back navigation
   - `composable<DashboardRoute>` -- placeholder Box with "Dashboard" text and navigate to ProfileRoute
   - `composable<ProfileRoute>` -- placeholder Box with "Profile" text and back navigation
4. For the LoginRoute -> DashboardRoute transition, use `popUpTo<LoginRoute> { inclusive = true }` to clear the auth back stack (see Pitfall 7 in research)

Use `BasicText` from Foundation (not Material3 `Text`). For placeholder screens, simple `Box(Modifier.fillMaxSize(), contentAlignment = Alignment.Center)` with `Column` containing `BasicText` and `Box` acting as clickable buttons. Style with hardcoded colors for now -- these placeholders will be replaced with real screens in Phase 5.

Import `androidx.navigation.compose.NavHost`, `androidx.navigation.compose.composable`, `androidx.navigation.compose.rememberNavController`.

**App.kt update:**
Replace the current App() composable content with `AppNavHost()`. Keep the existing `KoinApplication` wrapper. The structure should be:
```
KoinApplication(application = { modules(allAppModules) }) {
    AppNavHost()
}
```
Remove any Material3 theme wrappers (MaterialTheme) from App.kt if present -- the TerminalTheme will be added in plan 04-02.

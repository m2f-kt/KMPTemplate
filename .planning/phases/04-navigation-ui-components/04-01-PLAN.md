  <action>
    **Routes.kt** -- Create `composeApp/src/commonMain/kotlin/com/m2f/template/navigation/Routes.kt`:

    Define all route objects as @Serializable data objects or data classes in the `com.m2f.template.navigation` package:
    - `LoginRoute` -- data object (no args, auth entry point)
    - `RegisterRoute` -- data object (no args)
    - `DashboardRoute` -- data object (no args, main screen after auth)
    - `ProfileRoute` -- data class with `val userId: String? = null` (null = current user)

    All routes use `@Serializable` from `kotlinx.serialization`. No string-based route names anywhere.

    **AppNavHost.kt** -- Create `composeApp/src/commonMain/kotlin/com/m2f/template/navigation/AppNavHost.kt`:

    Create a `@Composable fun AppNavHost()` that:
    1. Creates `val navController = rememberNavController()`
    2. Defines a `NavHost` with `navController` and `startDestination = LoginRoute`
    3. Registers routes using `composable<T>` type-safe builders:
       - `composable<LoginRoute>` -- renders a placeholder `Text("Login Screen")` inside a `Box(Modifier.fillMaxSize(), contentAlignment = Center)`. Adds an `onLoginSuccess` button that navigates to `DashboardRoute` with `popUpTo<LoginRoute> { inclusive = true }` (see Pitfall 2 in research -- prevents back-to-login after auth).
       - `composable<RegisterRoute>` -- renders placeholder `Text("Register Screen")` with back navigation.
       - `composable<DashboardRoute>` -- renders placeholder `Text("Dashboard Screen")`.
       - `composable<ProfileRoute>` -- extracts route via `backStackEntry.toRoute<ProfileRoute>()` and renders placeholder showing userId.

    These are placeholder screens that Phase 5 will replace with real implementations from `app:auth` and `app:dashboard` feature modules. The navigation graph structure (routes, transitions, back stack management) is the deliverable here.

    Import `androidx.navigation.compose.NavHost`, `androidx.navigation.compose.composable`, `androidx.navigation.compose.rememberNavController` from the JetBrains navigation artifact (NOT `androidx.navigation`).

    **App.kt** -- Replace the demo content (AnimatedVisibility, Greeting, Image) with `AppNavHost()`:

    The updated App.kt should:
    1. Keep the `KoinApplication` wrapper with `allAppModules`
    2. Keep `MaterialTheme` wrapper (Plan 04-02 will replace this with `AppTheme`)
    3. Replace everything inside MaterialTheme with `AppNavHost()`
    4. Remove unused imports (AnimatedVisibility, Image, painterResource, mutableStateOf, remember, Res.drawable, etc.)

    Do NOT remove the `@Preview` annotation or the `@Composable` annotation on `App()`.

    All files in package `com.m2f.template.theme`.

    **Color.kt:**
    Define `LightColorScheme` and `DarkColorScheme` using `lightColorScheme()` and `darkColorScheme()` from Material3. Use the default Material3 purple/violet palette as the starting point (see research Pattern 4 for exact values). Include all 29 color roles (primary, onPrimary, primaryContainer, onPrimaryContainer, secondary, onSecondary, secondaryContainer, onSecondaryContainer, tertiary, onTertiary, tertiaryContainer, onTertiaryContainer, error, onError, errorContainer, onErrorContainer, background, onBackground, surface, onSurface, surfaceVariant, onSurfaceVariant, outline, outlineVariant, inverseSurface, inverseOnSurface, inversePrimary, scrim, surfaceTint).

    Add a comment at the top: `// Brand colors -- edit these to customize the app theme` before the color definitions. This makes it obvious to a developer cloning the template where to change colors.

    **Type.kt:**
    Define `val AppTypography = Typography(...)` using Material3 defaults (system fonts). Do NOT use custom fonts (see research Open Question 2 -- keep template lightweight). Define all major text styles with explicit values so developers see the structure and can customize:
    - `displayLarge`, `displayMedium`, `displaySmall`
    - `headlineLarge`, `headlineMedium`, `headlineSmall`
    - `titleLarge`, `titleMedium`, `titleSmall`
    - `bodyLarge`, `bodyMedium`, `bodySmall`
    - `labelLarge`, `labelMedium`, `labelSmall`

    Use `TextStyle(fontWeight = ..., fontSize = ..., lineHeight = ..., letterSpacing = ...)` for each. Use the standard Material3 type scale values. Add a comment explaining how to add custom fonts (reference Pitfall 4 from research about @Composable Font() in CMP).

    **Shape.kt:**
    Define `val AppShapes = Shapes(...)` with custom corner radius values using `RoundedCornerShape`:
    - `extraSmall = RoundedCornerShape(4.dp)`
    - `small = RoundedCornerShape(8.dp)`
    - `medium = RoundedCornerShape(12.dp)`
    - `large = RoundedCornerShape(16.dp)`
    - `extraLarge = RoundedCornerShape(28.dp)`

    **Theme.kt:**
    Create `@Composable fun AppTheme(darkTheme: Boolean = isSystemInDarkTheme(), content: @Composable () -> Unit)` that:
    1. Selects `LightColorScheme` or `DarkColorScheme` based on `darkTheme`
    2. Calls `MaterialTheme(colorScheme = colorScheme, typography = AppTypography, shapes = AppShapes, content = content)`

    Import `isSystemInDarkTheme` from `androidx.compose.foundation`.

    **App.kt update:**
    Replace `MaterialTheme { ... }` with `AppTheme { ... }`. Import `com.m2f.template.theme.AppTheme`. Remove the `import androidx.compose.material3.MaterialTheme` if it's no longer used directly (it may still be used indirectly -- check before removing).

  <action>
    All files in package `com.m2f.template.ui.components`. These are thin wrappers around Material3 components that enforce the template's design system defaults. Components must use `MaterialTheme.colorScheme.*` for colors -- ZERO hardcoded `Color(0x...)` values in any component file.

    **AppButton.kt:**
    Create 4 button composables (see research Pattern 5):

    1. `@Composable fun AppButton(text: String, onClick: () -> Unit, modifier: Modifier = Modifier, enabled: Boolean = true, loading: Boolean = false)` -- Primary filled button. When `loading = true`, shows a `CircularProgressIndicator` (size 18.dp, strokeWidth 2.dp, color `MaterialTheme.colorScheme.onPrimary`) next to the text and disables clicks. Uses `Button` from Material3.

    2. `@Composable fun AppTonalButton(text: String, onClick: () -> Unit, modifier: Modifier = Modifier, enabled: Boolean = true)` -- Uses `FilledTonalButton` from Material3.

    3. `@Composable fun AppOutlinedButton(text: String, onClick: () -> Unit, modifier: Modifier = Modifier, enabled: Boolean = true)` -- Uses `OutlinedButton` from Material3.

    4. `@Composable fun AppTextButton(text: String, onClick: () -> Unit, modifier: Modifier = Modifier, enabled: Boolean = true)` -- Uses `TextButton` from Material3.

    All buttons accept a `modifier` parameter and render the text via `Text(text)` as the button content.

    **AppTextField.kt:**
    Create `@Composable fun AppTextField(value: String, onValueChange: (String) -> Unit, label: String, modifier: Modifier = Modifier, error: String? = null, enabled: Boolean = true, singleLine: Boolean = true, keyboardOptions: KeyboardOptions = KeyboardOptions.Default, visualTransformation: VisualTransformation = VisualTransformation.None)`:
    - Wraps `OutlinedTextField` from Material3
    - Sets `modifier = modifier.fillMaxWidth()` (full-width by default)
    - Sets `label = { Text(label) }`
    - Sets `isError = error != null`
    - Sets `supportingText = error?.let { { Text(it, color = MaterialTheme.colorScheme.error) } }`
    - Passes through `enabled`, `singleLine`, `keyboardOptions`, `visualTransformation`

    Also create `@Composable fun AppPasswordField(value: String, onValueChange: (String) -> Unit, label: String = "Password", modifier: Modifier = Modifier, error: String? = null, enabled: Boolean = true)`:
    - Wraps `AppTextField` with `visualTransformation = PasswordVisualTransformation()` and `keyboardOptions = KeyboardOptions(keyboardType = KeyboardType.Password)`
    - Adds a trailing icon toggle for password visibility using `IconButton` with `Icons.Default.Visibility` / `Icons.Default.VisibilityOff` (use `remember { mutableStateOf(false) }` for visibility state)

    **AppCard.kt:**
    Create `@Composable fun AppCard(modifier: Modifier = Modifier, onClick: (() -> Unit)? = null, content: @Composable ColumnScope.() -> Unit)`:
    - If `onClick` is null, uses `Card` from Material3 (non-clickable)
    - If `onClick` is provided, uses `Card(onClick = onClick)` (clickable card)
    - Sets `modifier = modifier.fillMaxWidth()`
    - Applies default padding inside the card: wraps content in a `Column(modifier = Modifier.padding(16.dp))` with the provided `content` lambda

    Also create `@Composable fun AppElevatedCard(modifier: Modifier = Modifier, content: @Composable ColumnScope.() -> Unit)`:
    - Uses `ElevatedCard` from Material3
    - Same padding pattern as AppCard

    **AppDialog.kt:**
    Create `@Composable fun AppDialog(title: String, text: String, onDismiss: () -> Unit, onConfirm: () -> Unit, confirmText: String = "Confirm", dismissText: String = "Cancel")`:
    - Uses `AlertDialog` from Material3
    - Sets `title = { Text(title) }`, `text = { Text(text) }`
    - Sets `confirmButton` to `AppTextButton(text = confirmText, onClick = onConfirm)` (uses the AppTextButton from this library)
    - Sets `dismissButton` to `AppTextButton(text = dismissText, onClick = onDismiss)`
    - Sets `onDismissRequest = onDismiss`

    Also create `@Composable fun AppInfoDialog(title: String, text: String, onDismiss: () -> Unit, dismissText: String = "OK")`:
    - Single-action dialog for informational messages
    - Uses `AlertDialog` with only a confirm button (the dismiss button)
    - No confirm action, just dismissal

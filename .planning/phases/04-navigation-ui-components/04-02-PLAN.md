**TerminalTheme.kt** (`com.m2f.template.designsystem.theme`):
1. `@Composable fun TerminalTheme(darkTheme: Boolean = isSystemInDarkTheme(), content: @Composable () -> Unit)`:
   - Select colors: `val colors = if (darkTheme) TerminalDarkColors else TerminalLightColors`
   - Call `val typography = terminalTypography()` (composable because Font() is @Composable)
   - Wrap content in `CompositionLocalProvider` providing all 8 locals:
     - `LocalTerminalColors provides colors`
     - `LocalTerminalTypography provides typography`
     - `LocalTerminalSpacing provides TerminalSpacing(xs = 4.dp, sm = 8.dp, md = 12.dp, lg = 16.dp, xl = 20.dp)`
     - `LocalTerminalGap provides TerminalGap(xs = 4.dp, sm = 8.dp, md = 12.dp, lg = 16.dp, xl = 24.dp)`
     - `LocalTerminalShadows provides TerminalShadowValues`
     - `LocalTerminalOpacity provides TerminalOpacityValues`
     - `LocalTerminalRadius provides TerminalRadiusValues`
     - `LocalTerminalBorders provides TerminalBorderValues`

2. `object TerminalTheme` accessor with `@Composable get()` properties for all 8 subsystems:
   - `val colors: TerminalColors @Composable get() = LocalTerminalColors.current`
   - `val typography: TerminalTypography @Composable get() = LocalTerminalTypography.current`
   - `val spacing: TerminalSpacing @Composable get() = LocalTerminalSpacing.current`
   - `val gap: TerminalGap @Composable get() = LocalTerminalGap.current`
   - `val shadows: TerminalShadows @Composable get() = LocalTerminalShadows.current`
   - `val opacity: TerminalOpacity @Composable get() = LocalTerminalOpacity.current`
   - `val radius: TerminalRadius @Composable get() = LocalTerminalRadius.current`
   - `val borders: TerminalBorders @Composable get() = LocalTerminalBorders.current`

Note: The TerminalTheme composable function and the TerminalTheme object have the same name. This is an intentional Kotlin pattern used by Material3 (MaterialTheme function + MaterialTheme object). The compiler distinguishes them by usage context.

For `isSystemInDarkTheme()`: import from `androidx.compose.foundation.isSystemInDarkTheme`. If the import is not available from Foundation alone, try `compose.material3` temporarily or define a `expect fun isSystemInDarkTheme(): Boolean` with actual implementations per platform as a last resort. The research suggests Foundation includes it.

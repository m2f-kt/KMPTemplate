       ```kotlin
       /**
        * Delete an invitation record permanently.
        * Used when resending to avoid duplicate rows for the same email.
        * Returns true if a row was deleted.
        */
       suspend fun deleteById(id: Uuid): Boolean = suspendTransaction(db = db) {
           val rowsDeleted = InvitationsTable.deleteWhere { InvitationsTable.id eq id }
           rowsDeleted > 0
       }
       ```
       Add import for `org.jetbrains.exposed.v1.r2dbc.deleteWhere` (or the correct Exposed R2DBC delete import — check existing imports in the project for the pattern).

    2. **InvitationService.kt** — In `resendInvitation()` (lines 260-318), replace the revoke-old pattern with delete-old:
       - Remove the `if (oldInvitation.revokedAt == null) { invitationRepository.revokeById(iid) }` block (lines 291-293)

       - This ensures only one invitation per email exists in the table after resend
       - The `isExpiredOrRevoked` guard (line 284-288) still correctly prevents resending active invitations

    This is a clean fix: resend = delete old + create new (instead of revoke old + create new). The old invitation data is not needed for audit after resend since a brand new invitation is created.

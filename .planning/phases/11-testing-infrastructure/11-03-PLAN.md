    **FakeAuthApiBuilder.kt:**
    Package: `com.m2f.template.core.testing.fakes`

    1. Create `@FakeSDKDsl class FakeAuthApiBuilder` with private mutable lambda fields for all 6 AuthApi methods. Each defaults to returning `Either.Left(AppError.Client.Unknown())`:
       - `_register: suspend (RegisterRequest) -> Either<AppError, AuthResponse>`
       - `_login: suspend (LoginRequest, Boolean) -> Either<AppError, AuthResponse>`
       - `_refresh: suspend (RefreshTokenRequest) -> Either<AppError, AuthResponse>`
       - `_forgotPassword: suspend (ForgotPasswordRequest) -> Either<AppError, Unit>`
       - `_resetPassword: suspend (ResetPasswordRequest) -> Either<AppError, Unit>`
       - `_logout: suspend () -> Either<AppError, Unit>`

    2. Public setter methods (same names as interface methods) that replace the default lambda:
       - `fun register(behavior: suspend (RegisterRequest) -> Either<AppError, AuthResponse>)`
       - `fun login(behavior: suspend (LoginRequest, Boolean) -> Either<AppError, AuthResponse>)`
       - `fun refresh(behavior: suspend (RefreshTokenRequest) -> Either<AppError, AuthResponse>)`
       - `fun forgotPassword(behavior: suspend (ForgotPasswordRequest) -> Either<AppError, Unit>)`
       - `fun resetPassword(behavior: suspend (ResetPasswordRequest) -> Either<AppError, Unit>)`
       - `fun logout(behavior: suspend () -> Either<AppError, Unit>)`

    3. `internal fun build(): AuthApi` returns `object : AuthApi { ... }` delegating each method to its lambda.

    4. Top-level DSL entry: `fun fakeAuthApi(block: FakeAuthApiBuilder.() -> Unit = {}): AuthApi = FakeAuthApiBuilder().apply(block).build()`

    **FakeUserApiBuilder.kt:**
    Package: `com.m2f.template.core.testing.fakes`

    Same pattern for UserApi's 3 methods:
    - `_getProfile: suspend () -> Either<AppError, UserResponse>`
    - `_updateProfile: suspend (UpdateProfileRequest) -> Either<AppError, UserResponse>`
    - `_getUserById: suspend (String) -> Either<AppError, UserResponse>`

    Top-level: `fun fakeUserApi(block: FakeUserApiBuilder.() -> Unit = {}): UserApi = FakeUserApiBuilder().apply(block).build()`

    **NOTE:** `@FakeSDKDsl` annotation is already defined in `Annotations.kt` from Plan 02. Import it.

    **IMPORTANT:** These fakes live in core:testing `commonMain` (not `commonTest`), so they are available to any module that adds `testImplementation(projects.core.testing)`.

    **LoginViewModelTest.kt:**
    Package: `com.m2f.template.app.auth`

    Since LoginViewModel is not yet MVI (it uses old-style StateFlow), we test it with Turbine directly (not the MVI DSL). This proves the core:testing toolkit works. The MVI DSL will be exercised after Phase 12 migration.

    ```kotlin
    class LoginViewModelTest : ViewModelTest() {

        @Test
        fun `login success updates state with loginSuccess true`() = runTest {
            val authApi = fakeAuthApi {
                login { _, _ ->
                    Either.Right(AuthResponse(accessToken = "test-access", refreshToken = "test-refresh"))
                }
            }
            val viewModel = LoginViewModel(authApi)

            viewModel.state.test {
                // Initial state
                awaitItem() shouldBe LoginState()

                // Enter email
                viewModel.onEmailChange("user@test.com")
                awaitItem().email shouldBe "user@test.com"

                // Enter password
                viewModel.onPasswordChange("password123")
                awaitItem().password shouldBe "password123"

                // Trigger login
                viewModel.login()
                // Loading state
                val loading = awaitItem()
                loading.isLoading shouldBe true

                // Success state
                val success = awaitItem()
                success.loginSuccess shouldBe true
                success.isLoading shouldBe false

                cancelAndIgnoreRemainingEvents()
            }
        }

        @Test
        fun `login with blank email shows email error`() = runTest {
            val authApi = fakeAuthApi()  // unconfigured -- won't be called
            val viewModel = LoginViewModel(authApi)

            viewModel.state.test {
                awaitItem() // initial state

                viewModel.login()
                val errorState = awaitItem()
                errorState.emailError shouldBe "Email must not be blank"

                cancelAndIgnoreRemainingEvents()
            }
        }

        @Test
        fun `login failure shows server error`() = runTest {
            val authApi = fakeAuthApi {
                login { _, _ ->
                    Either.Left(AppError.Auth.InvalidCredentials())
                }
            }
            val viewModel = LoginViewModel(authApi)

            viewModel.state.test {
                awaitItem() // initial state

                viewModel.onEmailChange("user@test.com")
                awaitItem()
                viewModel.onPasswordChange("password123")
                awaitItem()

                viewModel.login()
                val loading = awaitItem()
                loading.isLoading shouldBe true

                val error = awaitItem()
                error.serverError shouldBe "Invalid credentials"
                error.isLoading shouldBe false

                cancelAndIgnoreRemainingEvents()
            }
        }
    }
    ```

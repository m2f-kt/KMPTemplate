    **Statement.kt:**
    ```kotlin
    sealed interface Statement<out Intent, out Model, out Event> {
        data class IntentStatement<Intent>(val intent: Intent) : Statement<Intent, Nothing, Nothing>
        data class ModelStatement<Model>(val expected: Model) : Statement<Nothing, Model, Nothing>
        data class EventStatement<Event>(val expected: Event) : Statement<Nothing, Nothing, Event>
    }
    ```

    **ViewModelTestContext.kt:**
    ```kotlin
    @ViewModelTestDsl
    class ViewModelTestContext<Intent, Model, Mutation, Event> {
        internal val statements = mutableListOf<Statement<Intent, Model, Event>>()

        fun intent(intent: Intent) {
            statements.add(Statement.IntentStatement(intent))
        }

        fun model(expected: Model) {
            statements.add(Statement.ModelStatement(expected))
        }

        fun event(expected: Event) {
            statements.add(Statement.EventStatement(expected))
        }
    }
    ```

    **MviViewModelTestDsl.kt:**
    Two extension functions on `MviViewModel<Intent, Model, Mutation, Event>`:

    1. `fun MviViewModel.test(context: CoroutineContext = EmptyCoroutineContext, block: ViewModelTestContext.() -> Unit)`:
       - Creates `ViewModelTestContext`, applies `block` to populate statements list
       - Inside `runTest(context)` + `turbineScope`:
         - `val modelTurbine = model.testIn(backgroundScope)`
         - `val eventTurbine = event.testIn(backgroundScope)`
         - `modelTurbine.awaitItem()` to skip initial StateFlow emission
         - Loop over statements: IntentStatement -> call `take(intent)`, ModelStatement -> `modelTurbine.awaitItem() shouldBe expected`, EventStatement -> `eventTurbine.awaitItem() shouldBe expected`
         - `modelTurbine.cancelAndIgnoreRemainingEvents()`
         - `eventTurbine.cancelAndIgnoreRemainingEvents()`

    2. `fun MviViewModel.scopedTest(context: CoroutineContext = EmptyCoroutineContext, block: suspend ViewModelTestContext.() -> Unit)`:
       - Same as `test` but `block` is `suspend` -- allows coroutine operations in test setup.

    Include a KDoc comment with usage example (the example from RESEARCH.md code_examples section).
    NO println statements anywhere. Clean test output only.

    **ViewModelTest.kt:**
    ```kotlin
    abstract class ViewModelTest {
        private val testDispatcher = StandardTestDispatcher()

        @BeforeTest
        fun setUp() {
            Dispatchers.setMain(testDispatcher)
        }

        @AfterTest
        fun tearDown() {
            Dispatchers.resetMain()
        }
    }
    ```
    Import from `kotlinx.coroutines.test.StandardTestDispatcher`, `setMain`, `resetMain` and `kotlin.test.BeforeTest`, `kotlin.test.AfterTest`.

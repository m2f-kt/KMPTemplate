# Phase 18.1: Profile Uploads, Group Creation & Email Invitations - Context

**Gathered:** 2026-02-22
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase delivers three connected capabilities building on Phase 18's file upload and email services:
1. Profile picture uploads from the UI with platform-compliant crop dialog
2. Group creation from the admin panel (admins only)
3. Email invitations to join groups with registration flow and deeplink support

</domain>

<decisions>
## Implementation Decisions

### Profile picture handling
- Picker with crop dialog, platform-compliant (Android, iOS, desktop, web)
- Client-side cropping before upload — server stores as-is
- Initials fallback when no profile picture (existing TerminalAvatar behavior)
- Upload flow accessible from Profile page only

### Group creation flow
- Admins only can create groups
- Name only required field — minimal form
- Group creation in admin panel
- Simple modal/inline form — no wizard

### Invitation mechanics
- Single invite at a time (no bulk)
- 7 day expiration for invitation links
- Default member role — admin promotes later
- Invite link opens registration form (not auto-create):
  - Mobile: deeplink opens app
  - Web: opens registration in browser
- Token carried in URL query param to registration endpoint
- Registration endpoint auto-assigns user to group from invite token

### Empty/error states
- Expired invite: "Expired invitation" message + link to request new invite
- Empty groups list: "No groups yet" empty state with create button
- Upload failure: show error with retry option
- Pending invitations: list with resend and revoke actions visible to admins

### Claude's Discretion
- Exact crop dialog UI implementation per platform
- Image size limits and file type validation
- Email template design
- Deeplink URL scheme

</decisions>

## Specific Ideas

- Platform-native feel for mobile (deeplink to app)
- Admin sees and manages pending invitations with actions

## Pre-Test Fixes

1. **AWS SDK version conflict** — Server failed to start with `NoClassDefFoundError: AwsSdkClientConfig$Builder`. Cause: Koog 0.6.2 brings `bedrockruntime:1.5.123` (uses `aws-core:1.5.123`) but S3 SDK was `1.4.45`. Fixed by upgrading `aws-s3` from `1.4.45` to `1.5.123`. Committed: `d66bc51`.

2. **Missing avatar_url column** — Login failed with "column users.avatar_url does not exist". The column wasn't created by Exposed auto-migration. Fixed manually with `ALTER TABLE users ADD COLUMN avatar_url VARCHAR(512)`.

3. **WASM deprecated asDynamic()** — Compilation warning about deprecated `asDynamic()`. Fixed by using typed `Uint8Array` with proper imports and `@OptIn(ExperimentalWasmJsInterop::class)`. Committed: `4405d63`.

4. **MinIO bucket policy missing public access** — Uploads succeeded (201 Created) but returned URLs were inaccessible (403 Access Denied). Root cause: the `uploads` bucket was missing the anonymous download policy. Fixed by adding `mc anonymous set download local/uploads` to docker-compose.yml minio-init service. Committed: `f556a73`.

5. **CORS missing localhost:8082** — WASM dev server started on port 8082 which wasn't in CORS allowlist. Fixed by adding `localhost:8082` to allowed origins. Committed: `19674f0`.

6. **Crop dialog image preview** — Crop dialog showed file size text instead of actual image. Created multiplatform `rememberDecodedImage` expect/actual function to decode ByteArray to ImageBitmap across all platforms. Committed: `93a3792`.

7. **Dashboard avatars not updating** — Dashboard sidebar and header showed hardcoded initials instead of using TerminalAvatar component. Also, navigating back from Profile didn't refresh avatar. Fixed by:
   - Adding `avatarUrl` to DashboardModel/Mutation/ViewModel
   - Using TerminalAvatar in DashboardSidebar and DashboardScreen
   - Using LifecycleEventObserver to trigger RefreshProfile on ON_RESUME
   - Removing redundant avatar from ProfileSidebar
   Commits: `b714c88`, `90e7f3f`, `d2d6683`.

8. **Multi-group UI hidden** — Creating additional groups when one exists caused UX issues (couldn't navigate between groups). Fixed by hiding "Create Group" button when a group already exists. Multi-group support deferred to future iteration.

9. **Missing invitations table** — Server returned 500 with exposed SQL error "relation invitations does not exist". Two fixes applied:
   - Manually created `invitations` table with all columns and indexes
   - Added StatusPages global exception handler to mask internal error details

9. **Invite link deep linking missing** — Two issues with invitation links:
   - APP_URL defaulted to port 8080 (server) instead of 8081 (web app)
   - WASM app had no deep link handling for `/invite/accept?token=...` URLs
   Fixed by:
   - Corrected APP_URL default to `http://localhost:8081`
   - Added `InviteLinkChecker` expect/actual (like `OAuthCallbackChecker`)
   - Added invite link detection in `App.kt` startup
   Commit: `e742812`

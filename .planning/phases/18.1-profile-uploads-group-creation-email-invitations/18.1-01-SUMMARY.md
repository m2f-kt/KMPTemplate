## Decisions Made
- **Avatar route module placement:** Created AvatarRoutes.kt in main server module rather than server:auth due to circular dependency (server:files depends on server:auth, so auth cannot depend on files). This follows a "cross-module route pattern" where routes needing services from multiple modules are wired at the Application.kt level.
- **File validation:** Avatar endpoint validates that contentType starts with "image/" before accepting upload

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 4 - Architectural] Relocated avatar route to main server module**
- **Found during:** Task 2 (Avatar endpoint implementation)
- **Issue:** Plan specified adding avatar endpoint to UserRoutes.kt in server:auth module with FileService dependency. Adding server:files dependency to server:auth created circular dependency (server:files already depends on server:auth)
- **Fix:** Created new AvatarRoutes.kt in server/src/main/kotlin/com/m2f/template/routes/ where both UserRepository and FileService are available via Koin injection at Application.kt
- **Files created:** server/src/main/kotlin/com/m2f/template/routes/AvatarRoutes.kt

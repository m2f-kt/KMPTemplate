# Phase 18.1 Plan 05: Invitation UI Summary

Admin panel invite dialog and invite acceptance screen with navigation route

## One-liner

Email invitation UI: admin sends invites via modal dialog, recipients accept via token-based InviteAcceptScreen

## Implementation Details

### Task 1: Admin Panel Invite Dialog (commit e1711ae)

Extended AdminPanelScreen with invitation functionality:

**Model additions:**
- `showInviteDialog`, `inviteEmail`, `isSendingInvite`, `inviteError`, `inviteSuccess`, `inviteLink`

**Intents added:**
- `OpenInviteDialog`, `CloseInviteDialog`, `InviteEmailChanged`, `SendInvite`

**ViewModel logic:**
- `handleSendInvite()` validates email, calls `sdk.createInvitation(groupId, CreateInvitationRequest(email))`
- Builds invite link from returned token on success

**Screen UI:**
- "Invite Member" button added next to "Register Member" button
- `InviteDialog` composable: email input → send → shows success with selectable invite link

### Task 2: InviteAcceptScreen + Navigation (commit 0bb69c7)

Created complete MVI stack for invitation acceptance:

**InviteAcceptModel:**
- Token, groupName, inviterName, role, email, isExpired, isAlreadyAccepted, error, acceptSuccess

**InviteAcceptViewModel:**
- `handleLoadInvitation(token)` - calls `sdk.getInvitation(token)`
- `handleAcceptInvitation()` - calls `sdk.acceptInvitation(AcceptInvitationRequest(token))`
- Emits `NavigateToGroup` event on success

**InviteAcceptScreen:**
- Shows loading state while fetching invitation
- Displays group name, inviter name, role
- Handles expired/already-accepted states with appropriate alerts
- Accept button triggers invitation acceptance

**Navigation:**
- Added `InviteAcceptRoute(token: String)` to Routes.kt
- Wired up in AppNavHost with event collection for navigation
- Registered `InviteAcceptViewModel` in AppModule
- On success: navigates to `AdminPanelRoute(groupId)`

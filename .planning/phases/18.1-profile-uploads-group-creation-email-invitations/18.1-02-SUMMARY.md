# Phase 18.1 Plan 02: Profile Avatar UI with Image Picker Summary

Profile avatar upload UI with Coil image loading, platform-specific ImagePicker expect/actual, and CropDialog integration in ProfileScreen.

## What Was Built

### Task 1: Coil + ImagePicker + TerminalAvatar Image Support

**Coil Integration:**
- Added Coil 3.2.0 to `libs.versions.toml` with coil-compose and coil-network-ktor
- Updated `app/designsystem/build.gradle.kts` with Coil dependencies

**TerminalAvatar Enhancement:**
- Added optional `imageUrl: String?` parameter
- Uses Coil `AsyncImage` with circular clip and ContentScale.Crop
- Falls back to initials display on error or when no URL provided
- Maintains existing initials-based rendering as fallback

**ImagePicker Expect/Actual:**
- Created `ImagePickerResult` data class with bytes, fileName, mimeType
- `rememberImagePickerLauncher` expect function returning launch callback

Platform implementations:
- **Android:** Uses `ActivityResultContracts.PickVisualMedia` with PickVisualMediaRequest
- **iOS:** Uses `PHPickerViewController` with `loadDataRepresentationForTypeIdentifier(UTTypeImage.identifier)`
- **JVM:** Uses Swing `JFileChooser` with image file filter
- **WASM:** Uses HTML `<input type="file">` with FileReader API

### Task 2: ProfileScreen Avatar Picker + Crop Dialog + ViewModel Upload

**ProfileModel State:**
- Added `avatarUrl: String?` for displaying current avatar
- Added `isUploadingAvatar: Boolean` for upload progress
- Added `showCropDialog: Boolean` for crop dialog visibility
- Added `pendingImageBytes: ByteArray?` for pre-upload image data
- Custom equals/hashCode for ByteArray comparison

**ProfileIntent Avatar Intents:**
- `ImageSelected(bytes: ByteArray, mimeType: String)` - when image picked
- `CropConfirmed` - user confirms upload
- `CropCancelled` - user cancels dialog

**ProfileMutation Avatar Mutations:**
- `SetPendingImage(bytes: ByteArray)` - store picked image
- `ShowCropDialog` / `HideCropDialog` - dialog visibility
- `SetUploadingAvatar(uploading: Boolean)` - upload state
- `SetAvatarUrl(url: String?)` - update avatar URL

**ProfileViewModel Handling:**
- `handleImageSelected` stores bytes and shows crop dialog
- `handleCropConfirmed` calls SDK uploadAvatar and updates avatarUrl
- `handleCropCancelled` hides dialog and clears pending bytes
- SetProfile mutation now includes avatarUrl from server response

**ProfileScreen UI:**
- ProfileHeader now includes 64dp tappable avatar with TerminalAvatar
- Upload progress overlay with TerminalProgress indicator
- "Tap to change avatar" hint text
- CropDialog composable with:
  - Full screen overlay with 90% opacity background
  - TerminalCard with title/description
  - Image size preview (MVP - actual crop manipulation deferred)
  - Confirm/Cancel buttons
- Image picker wired via `rememberImagePickerLauncher`

**Navigation Integration:**
- AppNavHost updated with new ProfileScreen callbacks:
  - `onImageSelected` → ProfileIntent.ImageSelected
  - `onCropConfirmed` → ProfileIntent.CropConfirmed
  - `onCropCancelled` → ProfileIntent.CropCancelled

    **Update build.gradle.kts:**
    Add `implementation(libs.arrow.core)`, `implementation(libs.androidx.lifecycle.viewmodelCompose)`, and `implementation(libs.koin.compose.viewmodel)` to app:auth commonMain dependencies. These are needed for Either, ViewModel, and koinViewModel.

    **LoginState.kt:**
    ```
    data class LoginState(
        val email: String = "",
        val password: String = "",
        val rememberMe: Boolean = false,
        val isLoading: Boolean = false,
        val loginSuccess: Boolean = false,
        val emailError: String? = null,
        val passwordError: String? = null,
        val serverError: String? = null,
    )
    ```

    **LoginViewModel.kt:**
    - Constructor: `class LoginViewModel(private val authApi: AuthApi) : ViewModel()`
    - StateFlow: `private val _state = MutableStateFlow(LoginState())` / `val state = _state.asStateFlow()`
    - Methods:
      - `fun onEmailChange(email: String)` -- updates email, clears emailError if previously set
      - `fun onPasswordChange(password: String)` -- updates password, clears passwordError if previously set
      - `fun onRememberMeChange(checked: Boolean)` -- updates rememberMe
      - `fun login()` -- validates email (non-empty, basic format) and password (non-empty) locally first. If validation fails, sets field errors. If validation passes, sets isLoading=true, clears serverError, calls `authApi.login(LoginRequest(email, password))`. Use `.fold(ifLeft = { set serverError from error.message, isLoading=false }, ifRight = { set loginSuccess=true, isLoading=false })`.
    - Validation approach (per discretion): validate on submit. Clear individual field errors on change.

    **RegisterState.kt:**
    ```
    data class RegisterState(
        val firstName: String = "",
        val lastName: String = "",
        val email: String = "",
        val password: String = "",
        val confirmPassword: String = "",
        val termsAccepted: Boolean = false,
        val isLoading: Boolean = false,
        val registerSuccess: Boolean = false,
        val fieldErrors: Map<String, String> = emptyMap(),
        val serverError: String? = null,
    )
    ```

    **RegisterViewModel.kt:**
    - Constructor: `class RegisterViewModel(private val authApi: AuthApi) : ViewModel()`
    - StateFlow pattern same as LoginViewModel.
    - Field change methods: `onFirstNameChange`, `onLastNameChange`, `onEmailChange`, `onPasswordChange`, `onConfirmPasswordChange`, `onTermsAcceptedChange` -- each clears its own field error from the fieldErrors map.
    - `fun register()`:
      1. Use Arrow `either { zipOrAccumulate(...) }` to validate ALL fields at once:
         - validateName(firstName) mapped to field "firstName"
         - validateName(lastName) mapped to field "lastName"
         - validateEmail(email) mapped to field "email"
         - validatePassword(password) mapped to field "password"
         - Ensure confirmPassword == password (field "confirmPassword", message "Passwords do not match")
         - Ensure termsAccepted (field "terms", message "You must accept the terms")
      2. On validation failure (Left): extract NonEmptyList<FieldError>, convert to Map<String, String> for fieldErrors
      3. On validation success (Right): set isLoading=true, call `authApi.register(RegisterRequest(email, password, firstName, lastName))`. Fold result: Left -> serverError, Right -> registerSuccess=true
    - Use the existing `validateEmail`, `validatePassword`, `validateName` from `core/models/validation/ValidationSupport.kt`. For confirmPassword and termsAccepted, write inline validation lambdas within the zipOrAccumulate block.
    - IMPORTANT: The validation functions use `context(raise: Raise<FieldError>)` pattern. Inside zipOrAccumulate these work naturally since each accumulation slot provides a Raise<FieldError> context.

    **ForgotPasswordState.kt:**
    ```
    data class ForgotPasswordState(
        val email: String = "",
        val isLoading: Boolean = false,
        val emailSent: Boolean = false,
        val emailError: String? = null,
        val serverError: String? = null,
    )
    ```

    **ForgotPasswordViewModel.kt:**
    - Constructor: `class ForgotPasswordViewModel(private val authApi: AuthApi) : ViewModel()`
    - Same StateFlow pattern.
    - `fun onEmailChange(email: String)` -- updates email, clears emailError
    - `fun submit()` -- validates email, calls authApi.forgotPassword, sets emailSent=true on success or serverError on failure

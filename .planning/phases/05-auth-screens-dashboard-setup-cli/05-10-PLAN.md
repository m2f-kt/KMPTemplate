<objective>
Restore ProfileRoute as a top-level navigation destination instead of an embedded composable slot inside DashboardScreen.

Purpose: The 05-08 gap closure embedded ProfileScreen inside DashboardScreen via composable slot injection, causing a double-sidebar on desktop (DashboardSidebar 260dp + ProfileSidebar 260dp) and doubled back-button headers on mobile. The existing `composable<ProfileRoute>` block in AppNavHost is already correct but orphaned -- nothing navigates to it. This plan removes all profile embedding from the dashboard and rewires profile clicks to use `navController.navigate(ProfileRoute)`.

Output: Profile renders as a full-screen standalone route. Dashboard state-based content switching (sidebar/bottom nav persistence for processes/logs/etc) remains untouched.
</objective>

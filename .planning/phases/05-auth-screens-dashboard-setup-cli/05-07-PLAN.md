    1. Add `OAUTH_REDIRECT_SCHEMES` config to Env.kt:
       - `OAUTH_WASM_REDIRECT_URL` (default: "http://localhost:8080/auth/callback") -- the WASM app URL
       - `OAUTH_MOBILE_SCHEME` (default: "template") -- custom URL scheme for deep links
       - `OAUTH_DESKTOP_LOCALHOST_PORT` (default: "9876") -- localhost port for desktop callback

    2. Update `OAuthRoutes.kt` callback handlers:
       The callback currently does `call.respond(authResponse)`. Change it to:
       - Read `redirect_uri` from query parameter (set by client when initiating OAuth)
       - Generate the AuthResponse (access token + refresh token) as before
       - Instead of `call.respond(authResponse)`, do:
         ```kotlin
         val redirectUri = call.request.queryParameters["state"]
             ?: env.oauthWasmRedirectUrl  // fallback to WASM redirect
         val separator = if ("?" in redirectUri) "&" else "?"
         call.respondRedirect(
             "$redirectUri${separator}access_token=${authResponse.accessToken}&refresh_token=${authResponse.refreshToken}"
         )
         ```
       - The `state` parameter is used to carry the client's redirect URI through the OAuth flow (Ktor's OAuth plugin passes it through). This is safe because the server is the one constructing the final JWT -- the redirect URI just controls where the browser goes after.
       - IMPORTANT: Validate the redirect URI against an allowlist of known patterns (WASM URL, custom scheme, localhost) before redirecting to prevent open redirect attacks.

       - After OAuth, browser redirects back to `/auth/callback?access_token=...&refresh_token=...`

       **androidMain/OAuthHandler.android.kt:**

         ```xml
           <action android:name="android.intent.action.VIEW" />
           <category android:name="android.intent.category.DEFAULT" />
           <category android:name="android.intent.category.BROWSABLE" />
           <data android:scheme="template" android:host="auth" android:pathPrefix="/callback" />
         </intent-filter>
         ```

       - `getRedirectUri()`: Returns `"template://auth/callback"` (universal link / custom scheme)
       - iOS universal links require Associated Domains entitlement -- for now use custom scheme which works without additional setup

       **jvmMain/OAuthHandler.jvm.kt:**

       - Additionally: start a temporary `ServerSocket(9876)` that listens for ONE request, extracts query params, then shuts down. Run in a coroutine. This is the standard desktop OAuth pattern.
       - IMPORTANT: The localhost server must be started BEFORE opening the browser URL, so the callback has somewhere to land.

    **Create OAuthCallbackHandler composable:**

    1. Create `app/auth/src/commonMain/.../OAuthCallbackHandler.kt`:
       ```kotlin
       @Composable
       fun OAuthCallbackHandler(
           accessToken: String,
           refreshToken: String,
           tokenStorage: TokenStorage,
           onSuccess: () -> Unit,
           onError: (String) -> Unit,
       ) {
           LaunchedEffect(accessToken, refreshToken) {
               try {
                   tokenStorage.saveTokens(accessToken, refreshToken)
                   onSuccess()
               } catch (e: Exception) {
                   onError(e.message ?: "Failed to save OAuth tokens")
               }
           }
       }
       ```
       This is a simple composable that stores the tokens and signals navigation.

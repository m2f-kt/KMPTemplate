    **Create app:profile module:**

    1. Create `app/profile/build.gradle.kts` mirroring app/auth and app/dashboard structure:
       - plugins: kmp-library-convention, com.android.library, composeMultiplatform, composeCompiler
       - KMP targets: android, ios, jvm, wasmJs
       - commonMain deps: projects.core.models, projects.core.sdk, projects.app.designsystem, compose.runtime, compose.foundation, libs.arrow.core, libs.androidx.lifecycle.viewmodelCompose, libs.koin.compose.viewmodel
       - android namespace: "com.m2f.template.app.profile"

    2. Add `include("app:profile")` to settings.gradle.kts under "App feature modules" comment.

    **ProfileState.kt:**
    ```kotlin
    data class ProfileState(
        val userId: String = "",
        val email: String = "",
        val name: String = "",
        val tier: UserTier = UserTier.Free,
        val isLoading: Boolean = true,
        val isEditing: Boolean = false,
        val editName: String = "",
        val editEmail: String = "",
        val fieldErrors: Map<String, String> = emptyMap(),
        val serverError: String? = null,
        val saveSuccess: Boolean = false,
        val logoutTriggered: Boolean = false,
    )
    ```

    **ProfileViewModel.kt:**
    - `class ProfileViewModel(private val userApi: UserApi, private val authApi: AuthApi) : ViewModel()`
    - `init { loadProfile() }` -- on creation, immediately fetch profile
    - `fun loadProfile()`: viewModelScope.launch, call `userApi.getProfile()`, fold:
      - Right: update state with user data, map `role` to `UserTier.fromString(role)`, isLoading=false
      - Left: set serverError, isLoading=false
    - `fun startEditing()`: set isEditing=true, copy current name/email to editName/editEmail
    - `fun cancelEditing()`: set isEditing=false, clear editName/editEmail/fieldErrors
    - `fun onEditNameChange(name: String)`: update editName, clear name fieldError
    - `fun onEditEmailChange(email: String)`: update editEmail, clear email fieldError
    - `fun saveProfile()`: validate editName/editEmail, call `userApi.updateProfile(UpdateProfileRequest(name=editName, email=editEmail))`, fold: Right -> update state, isEditing=false, saveSuccess=true; Left -> serverError
    - `fun logout()`: viewModelScope.launch, call `authApi.logout()` (result doesn't matter -- tokens always cleared), set logoutTriggered=true

    **ProfileScreen.kt:**
    Responsive layout with BoxWithConstraints:
    - Desktop (>840dp): Row with ProfileSidebar (260dp) + main content
    - Mobile: Column with back button header + content

    Main content uses `when (state.tier)` exhaustive match to render tier-specific content composable:
    ```kotlin
    when (state.tier) {
        is UserTier.Free -> FreeTierContent(state, ...)
        is UserTier.Paid -> PaidTierContent(state, ...)
        is UserTier.Premium -> PremiumTierContent(state, ...)
        is UserTier.Admin -> AdminTierContent(state, ...)
        is UserTier.PowerAdmin -> PowerAdminTierContent(state, ...)
    }
    ```

    If isEditing, show editable TerminalInputs for name and email with save/cancel buttons.
    If isLoading, show TerminalProgress indeterminate.

    **ProfileSidebar.kt** -- Tier-aware nav items per Pencil designs:
    - Free (EnD7X): 3 nav items (profile, preferences, billing) + upgrade card at bottom
    - Paid (ENKv0): 6 nav items + premium upgrade info
    - Premium (d4oDS): 7 nav items + premium success alert
    - Admin (uGWqD): 7 management + 2 tools nav items
    - PowerAdmin (QFwfO): CRM (6) + System (5) nav items
    - Use `when(tier)` to determine which nav items to show
    - Bottom: user info row + "$ logout" clickable link

    **Tier content composables (per Pencil designs):**

    - **FreeTierContent.kt**: Account info card (email, name, member since), usage limits card (API calls, storage -- mock data with TerminalProgress bars), warning alert "You are on the free tier", preferences section, locked features list (dimmed), upgrade CTA TerminalButton

    - **PaidTierContent.kt**: Account info (more fields), team access section, analytics preview, export options

    - **PremiumTierContent.kt**: Full features, webhook configuration, priority support card, API key display

    - **AdminTierContent.kt**: User management table, groups section, permissions matrix, analytics, audit log, org settings

    - **PowerAdminTierContent.kt**: Platform stats (uptime, users, requests), user directory, admin identity card, access matrix, system status, danger zone (destructive actions)

    Each tier content is a separate composable file taking `ProfileState` + callbacks. All use TerminalCard, TerminalTable, TerminalList, TerminalAlert, TerminalProgress, TerminalBadge, TerminalButton. **All tier-specific sections are demonstration/mock content with hardcoded static data** -- they show what each tier WOULD see but are not backed by real data sources. The ONLY functional profile operations are: (1) edit profile name and email via PUT /me, and (2) logout. Everything else (usage limits, team access, webhooks, API keys, analytics, audit log, admin tools, danger zone actions) is static display demonstrating the tier-differentiated UI.

           val viewModel = koinViewModel<ProfileViewModel>()

               state = state,
               onStartEditing = viewModel::startEditing,
               onCancelEditing = viewModel::cancelEditing,

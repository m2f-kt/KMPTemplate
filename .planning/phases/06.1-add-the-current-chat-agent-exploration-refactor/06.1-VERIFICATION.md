### Critical Patterns Verified

1. **Infinite loop fix:** The `processFrames forwardTo nodeFinish onCondition { it.filterIsInstance<Message.Tool.Call>().isEmpty() }` edge ensures text-only responses terminate the agent loop instead of looping back through `giveFeedbackToCallTools`.

2. **Per-request AIAgent pattern:** Each SSE/HTTP request creates a new `AIAgent` instance inside `callbackFlow`, allowing per-request strategy callback injection without singleton state issues.

3. **Streaming callback bridge:** `chatStreamingStrategy { frame -> trySend(frame.text) }` connects Koog's `StreamFrame.Append` events to Kotlin Flow emissions.

4. **SSE EventSource compatibility:** JWT validation from query parameter (`?token=JWT`) instead of Authorization header, enabling browser EventSource API usage.

5. **Typed SSE events:** Distinct event types (`conversation`, `message`, `done`, `error`) allow structured client-side event handling.

6. **Backward compatibility:** Existing `POST /api/ai/chat` endpoint preserved by collecting `streamChat()` flow in `run()` method.

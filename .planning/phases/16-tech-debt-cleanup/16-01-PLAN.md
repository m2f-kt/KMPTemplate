<objective>
Fix WASM locale persistence so the user's language selection survives page reloads.

Purpose: Currently, when a WASM user selects a locale (e.g., Spanish), refreshes the page, the locale resets to the browser default because `window.__customLocale` is not set before Compose Resources reads `navigator.languages`. The `multiplatform-settings` `StorageSettings` already writes the preference to `localStorage` under the key `com.russhwolf.settings.pref_language`, but nothing reads it before WASM boots.

Output: Two file changes that close the timing gap -- a pre-WASM localStorage read in `index.html` and a localStorage fallback in `getAppLocale()`.
</objective>

**index.html changes:**

Add a localStorage read BEFORE the existing `navigator.languages` monkey-patch script. The `multiplatform-settings` `StorageSettings` on WASM stores keys with the prefix `com.russhwolf.settings.`. The language key is `pref_language`, so the full localStorage key is `com.russhwolf.settings.pref_language`.

In the existing `<script>` block, BEFORE the line that reads `Object.getOwnPropertyDescriptor(Navigator.prototype, "languages")`, add:

```javascript
// Restore persisted locale from localStorage BEFORE WASM loads.
// multiplatform-settings StorageSettings uses "com.russhwolf.settings." prefix.
var savedLocale = localStorage.getItem("com.russhwolf.settings.pref_language");
if (savedLocale) {
    window.__customLocale = savedLocale;
}
```

This ensures `window.__customLocale` is already set when the monkey-patched `navigator.languages` getter is first called by Compose Resources during WASM initialization.

**AppLocale.wasmJs.kt changes:**

1. Add a new JS interop function `getLocalStorageLocale()` that reads directly from localStorage:

```kotlin
@OptIn(ExperimentalWasmJsInterop::class)
private fun getLocalStorageLocale(): JsString? =
    js("localStorage.getItem('com.russhwolf.settings.pref_language')")
```

2. Update `getAppLocale()` to include localStorage as a fallback between `overrideLocale` and `browserLanguage()`:

```kotlin
@OptIn(ExperimentalWasmJsInterop::class)
actual fun getAppLocale(): String =
    overrideLocale
        ?: getLocalStorageLocale()?.toString()
        ?: browserLanguage()
```

This creates a robust fallback chain: in-memory override (current session) -> localStorage (persisted across reloads) -> browser default (first visit).

**Important:** The `getLocalStorageLocale()` function MUST be a separate top-level function because Kotlin/WASM `js()` calls must be the sole expression in a top-level function body (established pattern from Phase 15, see `navigatorLanguage()`).

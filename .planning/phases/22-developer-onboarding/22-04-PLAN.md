- **Key Patterns** — Explain established patterns:
  - Context parameters for DI (`context(config: Configuration, database: R2dbcDatabase)`)
  - Koin module registration
  - Feature module structure (routes, service, repository, migrations)
  - Arrow Either for error handling
  - MVI ViewModel pattern (reference .claude/skills/mvi-viewmodel/)
- **Add a Feature** — Pattern reference guide (not scaffolding):
  1. Create module directory under `server/`
  2. Add `build.gradle.kts` with `server-module-convention` plugin
  3. Include in `settings.gradle.kts`
  4. Create routes, service, repository
  5. Register migrations via `MigrationRegistry.register()`
  6. Create Koin module
  7. Wire routes in Application.kt
  Point to auth and groups modules as canonical examples.

**2. Create `docs/GETTING-STARTED.md`:**

First-run walkthrough as markdown checklist per CONTEXT.md decisions:

- **Prerequisites** — JDK 11+, Docker Desktop, Git
- **Step 1: Clone** — `git clone` + `cd`
- **Step 2: Configure** — `cp .env.example .env` (optional — defaults work for local dev)
- **Step 3: Setup** — `./gradlew devSetup` (runs prerequisite checks + starts Docker)
- **Step 4: Seed** — `./gradlew seedData` (creates demo user: dev@example.com / password)
- **Step 5: Start Server** — `./gradlew :server:run`
- **Step 6: Verify** — `./gradlew verifySetup` OR manual checks:
  - `curl http://localhost:8080/health` — should return JSON with all services "up"
  - Visit http://localhost:8025 — MailHog web UI
  - Visit http://localhost:9003 — MinIO console (minioadmin/minioadmin)
  - `docker ps` — all containers running
- **Step 7: Start Web App** — `./gradlew :composeApp:wasmJsBrowserDevelopmentRun`
- **Step 8: Login** — Use dev@example.com / password

**Troubleshooting section** (embedded in same file, per CONTEXT.md):
Format: Problem → Cause → Fix (exact command)

Issues to cover:
- **Port already in use** → Another process using port → `lsof -ti:PORT | xargs kill`
- **Docker containers won't start** → Docker Desktop not running → Start Docker Desktop
- **Database connection refused** → PostgreSQL not healthy → `docker compose logs postgres`
- **Gradle build fails** → Wrong JDK version → Check `java -version`, install JDK 11+
- **Server won't start** → Missing migrations → Migrations run automatically on first start, check logs
- **MinIO bucket missing** → minio-init didn't run → `docker compose up minio-init`
- **Emails not appearing** → MailHog not running → Check `docker ps`, restart with `docker compose up -d mailhog`

1. **checkSetup** — Custom task that checks ALL prerequisites at once, reports all issues, then fails if any:
   ```kotlin
   tasks.register("checkSetup") {
       group = "dev"
       description = "Check development prerequisites"
       doLast {
           data class Check(val name: String, val passed: Boolean, val fix: String)
           val checks = mutableListOf<Check>()

           // Check Docker running
           val dockerResult = exec { commandLine("docker", "info"); isIgnoreExitValue = true }
           checks.add(Check("Docker running", dockerResult.exitValue == 0,
               "Install Docker Desktop: https://docs.docker.com/get-docker/"))

           // Check Docker Compose available
           val composeResult = exec { commandLine("docker", "compose", "version"); isIgnoreExitValue = true }
           checks.add(Check("Docker Compose available", composeResult.exitValue == 0,
               "Docker Compose is included with Docker Desktop. Update Docker Desktop if missing."))

           // Check JDK version (11+)
           val javaVersion = System.getProperty("java.version").split(".")[0].toIntOrNull() ?: 0
           checks.add(Check("JDK 11+ (found: ${System.getProperty("java.version")})", javaVersion >= 11,
               "Install JDK 11+: https://adoptium.net/"))

           // Check port availability for each Docker service port
           val ports = mapOf(5436 to "PostgreSQL", 9002 to "MinIO API", 9003 to "MinIO Console",
               1025 to "MailHog SMTP", 8025 to "MailHog Web", 8080 to "Server")
           for ((port, service) in ports) {
               val available = try { java.net.ServerSocket(port).use { true } } catch (_: Exception) { false }
               checks.add(Check("Port $port ($service) available", available,
                   "Port $port in use. Stop the process using it: lsof -ti:$port | xargs kill"))
           }

           // Print results
           println("\n  Development Environment Check\n")
           checks.forEach { check ->
               val icon = if (check.passed) "✅" else "❌"
               println("  $icon ${check.name}")
               if (!check.passed) println("     Fix: ${check.fix}")
           }
           println()

           val failures = checks.filter { !it.passed }
           if (failures.isNotEmpty()) {
               throw GradleException("${failures.size} prerequisite(s) failed. Fix the issues above and re-run ./gradlew checkSetup")
           }
           println("  All checks passed ✅\n")
       }
   }
   ```

   NOTE: Use `providers.exec {}` or ProcessBuilder instead of bare `exec {}` if Gradle version requires it. The exact API depends on Gradle version — use `ProcessBuilder` for reliability:
   ```kotlin
   fun runCommand(vararg args: String): Int {
       val process = ProcessBuilder(*args).redirectErrorStream(true).start()
       process.inputStream.bufferedReader().readLines() // consume output
       return process.waitFor()
   }
   ```

   - Password: bcrypt hash of "password" — pre-compute using jbcrypt: `$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy` (this is a standard bcrypt hash of "password")

   - Use INSERT ... ON CONFLICT DO NOTHING for idempotency
   - The users table uses UUID primary keys (kotlin.uuid.Uuid) — generate a fixed UUID for the seed user

   ```sql
   -- Seed data: demo user for development
   -- Credentials: dev@example.com / password
   -- Idempotent: skips if user already exists
   INSERT INTO users (id, email, name, password_hash, created_at)
   VALUES (
       '00000000-0000-0000-0000-000000000001',
       'dev@example.com',
       'Dev User',
       '$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy',
       NOW()
   )
   ON CONFLICT (email) DO NOTHING;
   ```

   IMPORTANT: Check the actual users table column names by reading the auth migration or UsersTable definition. Adjust column names to match exactly. Look in `server/auth/` for the table definition.

2. Add `seedData` Gradle task to `build.gradle.kts`:

   ```kotlin
       group = "dev"
       description = "Seed demo user (dev@example.com / password)"
       commandLine("docker", "exec", "-i", "template-postgres",
           "psql", "-U", "postgres", "-d", "application",

   ```

  <files>gradle/libs.versions.toml, server/core/config/build.gradle.kts, server/core/config/src/main/kotlin/com/m2f/core/config/configuration/Env.kt</files>
  <action>
1. Add dotenv-kotlin to version catalog:
   - In `gradle/libs.versions.toml` [versions]: `dotenv = "6.4.2"`
   - In [libraries]: `dotenv-kotlin = { module = "io.github.cdimascio:dotenv-kotlin", version.ref = "dotenv" }`

2. Add dependency to server/core/config/build.gradle.kts:
   - `implementation(libs.dotenv.kotlin)`

3. Modify Env.kt to use dotenv:
   - Add import: `import io.github.cdimascio.dotenv.Dotenv`
   - Add a private top-level dotenv instance:
     ```kotlin
     private val dotenv: Dotenv = Dotenv.configure()
         .ignoreIfMissing()
         .load()
     ```
   - Add a private helper function:
     ```kotlin
     private fun env(key: String): String? = dotenv[key] ?: System.getenv(key)
     ```
   - Replace ALL `System.getenv("X")` calls with `env("X")` throughout Env.kt (Http, Auth, OAuth, Ai, S3, Email sections)
   - Do NOT change any default values or the data class structure
   - The function signature of env() means: dotenv takes priority, System.getenv is fallback

Note: The dotenv library looks for `.env` in the working directory by default. When running via `./gradlew :server:run`, the working directory is the project root, so `.env` at project root will be found automatically.
  </action>

  </verify>

    2. MODIFY `composeApp/src/commonMain/kotlin/com/m2f/template/App.kt`:
    
    Current code:
    ```kotlin
    @Composable
    fun App() {
        KoinApplication(application = { modules(allAppModules) }) {
            val preferencesStorage = koinInject<PreferencesStorage>()
            val storedLocale = remember { preferencesStorage.language }
            LaunchedEffect(Unit) { setAppLocale(storedLocale) }
            TerminalTheme { AppNavHost() }
        }
    }
    ```

    Replace with:
    ```kotlin
    @Composable
    fun App() {
        KoinApplication(application = { modules(allAppModules) }) {
            val preferencesStorage = koinInject<PreferencesStorage>()
            val currentLocale by preferencesStorage.observeLanguage()
                .collectAsState(initial = preferencesStorage.language)
            
            LaunchedEffect(currentLocale) {
                setAppLocale(currentLocale)
            }
            
            CompositionLocalProvider(LocalAppLocale provides currentLocale) {
                key(currentLocale) {
                    TerminalTheme {
                        AppNavHost()
                    }
                }
            }
        }
    }
    ```

    Key changes:
    - `observeLanguage()` collected as Compose State (reactive)
    - `LaunchedEffect(currentLocale)` — re-runs setAppLocale when locale changes
    - `CompositionLocalProvider` — makes locale available to children
    - `key(currentLocale)` — forces full recomposition of entire UI tree when locale changes, which makes all stringResource() calls re-evaluate against the new Locale.getDefault()

    Add required imports:
    - `import androidx.compose.runtime.CompositionLocalProvider`
    - `import androidx.compose.runtime.collectAsState`
    - `import androidx.compose.runtime.getValue`
    - `import androidx.compose.runtime.key`
    - `import com.m2f.template.localization.LocalAppLocale`

    Remove the now-unnecessary `remember` import if no longer used.

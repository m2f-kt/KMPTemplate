2. **Create the bridge function** at `composeApp/src/commonMain/kotlin/com/m2f/template/localization/StringKeyResolver.kt`:

```kotlin
@Composable
fun stringResource(key: StringKey, vararg args: Any): String =
    when (key) {
        StringKey.AUTH_INVALID_CREDENTIALS -> org.jetbrains.compose.resources.stringResource(Res.string.error_auth_invalid_credentials, *args)
        StringKey.VALIDATION_EMAIL_BLANK -> org.jetbrains.compose.resources.stringResource(Res.string.error_validation_email_blank, *args)
        // ... exhaustive when for ALL StringKey entries
        StringKey.GENERIC_ERROR -> org.jetbrains.compose.resources.stringResource(Res.string.error_generic, *args)
    }
```

The `when` must be exhaustive over all StringKey entries. This provides compile-time checking — adding a new StringKey forces adding its resource mapping.

Import `org.jetbrains.compose.resources.stringResource` and disambiguate from our custom function by using the fully qualified name inside the function body, OR name our function `resolveStringKey` to avoid collision.

**Decision:** Name the bridge function `resolveStringKey(key: StringKey, vararg args: Any): String` to avoid name collision with the platform `stringResource()`.

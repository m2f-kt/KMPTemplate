            "VALIDATION_INVALID_INPUT" to "Entrada no válida",

    /**
     * Resolve a localized message for the given error code and locale.
     * Falls back to English, then to the raw code if no translation found.
     */
    fun resolve(code: String, locale: String): String {
        val lang = locale.take(2).lowercase()
        return strings[lang]?.get(code)

            ?: code

**1. Add locale extraction helper to Error.kt:**

Add a helper function to extract the preferred language from the Accept-Language header:

```kotlin
fun RoutingContext.preferredLanguage(): String =
    call.request.headers["Accept-Language"]
        ?.split(",")
        ?.firstOrNull()
        ?.trim()
        ?.take(2)
        ?.lowercase()
        ?: "en"
```

This parses `Accept-Language: es-ES,es;q=0.9,en;q=0.8` and extracts `"es"`. Falls back to `"en"` if header is missing.

**2. Refactor DomainError.respond() implementations in DomainError.kt:**

For each DomainError implementation (UnexpectedError, MappingError, IncorrectInput, MissingParameter, InvalidParameter, InvalidContent, Unauthorized), update the respond() method to resolve the message from ServerStrings:

```kotlin
// Before (UnexpectedError):
context(routingContext: RoutingContext)
override suspend fun respond() {
    val error = toAppError()
    routingContext.unexpected(error.code, error.message)
}

// After:
context(routingContext: RoutingContext)
override suspend fun respond() {
    val error = toAppError()
    val locale = routingContext.preferredLanguage()
    val message = ServerStrings.resolve(error.code, locale)
    routingContext.unexpected(error.code, message)
}
```

Apply this pattern to ALL DomainError implementations in DomainError.kt: UnexpectedError, MappingError, Unauthorized.

For IncorrectInput, MissingParameter, InvalidParameter, InvalidContent — these are validation errors with dynamic content. Keep the English format for dynamic parts but use ServerStrings for the error code message where applicable. For these, the `respond()` already constructs detailed messages — leave the validation detail messages as-is but resolve the base error code message from ServerStrings.

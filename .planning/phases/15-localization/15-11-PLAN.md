  <action>
    1. FIX WASM js() expression in AppLocale.wasmJs.kt:

    Current code (line 15-16):
    ```kotlin
    actual fun getAppLocale(): String =
        overrideLocale ?: js("navigator.language").toString().take(2)
    ```

    Kotlin/Wasm requires js() calls to be the ENTIRE body of a top-level function or a property initializer — they cannot be nested inside elvis expressions, if-else, or other expressions.

    Fix by extracting to a separate top-level function:
    ```kotlin
    private fun browserLanguage(): String = js("navigator.language").toString().take(2)

    actual fun getAppLocale(): String =
        overrideLocale ?: browserLanguage()
    ```

    This makes the js() call the entire body of a top-level function, satisfying the Kotlin/Wasm compiler requirement.

    2. ADD Android network security config for localhost cleartext:

    CREATE `composeApp/src/androidMain/res/xml/network_security_config.xml`:
    ```xml
    <?xml version="1.0" encoding="utf-8"?>
    <network-security-config>
        <domain-config cleartextTrafficPermitted="true">
            <domain includeSubdomains="false">localhost</domain>
            <domain includeSubdomains="false">10.0.2.2</domain>
            <domain includeSubdomains="false">10.0.3.2</domain>
        </domain-config>
    </network-security-config>
    ```

    10.0.2.2 = Android Emulator localhost alias
    10.0.3.2 = Genymotion localhost alias

    3. MODIFY `composeApp/src/androidMain/AndroidManifest.xml`:

    Add `android:networkSecurityConfig` and `android:usesCleartextTraffic` to the application tag:
    ```xml
    <application
            android:allowBackup="true"
            android:icon="@mipmap/ic_launcher"
            android:label="@string/app_name"
            android:roundIcon="@mipmap/ic_launcher_round"
            android:supportsRtl="true"
            android:usesCleartextTraffic="true"
            android:networkSecurityConfig="@xml/network_security_config"
            android:theme="@android:style/Theme.Material.Light.NoActionBar">
    ```

    Both attributes are needed: `usesCleartextTraffic` is the global flag, `networkSecurityConfig` provides the domain-specific policy.
  </action>

    1. MODIFY ApiClient.kt — add a localeProvider parameter and set Accept-Language header:

    Change the function signature to accept a locale provider:
    ```kotlin
    fun createApiClient(
        authInterceptor: AuthInterceptor,
        baseUrl: String = defaultBaseUrl(),
        localeProvider: () -> String = { "en" },
    ): HttpClient {
    ```

    In the defaultRequest block, add the Accept-Language header:
    ```kotlin
    defaultRequest {
        url(baseUrl)
        contentType(ContentType.Application.Json)
        header(HttpHeaders.AcceptLanguage, localeProvider())
    }
    ```

    Add required import:
    ```kotlin
    import io.ktor.http.HttpHeaders
    import io.ktor.client.request.header
    ```

    The localeProvider is a lambda (not a static value) because the locale can change at runtime. Each request will call the lambda to get the current locale.

    2. MODIFY SdkModule.kt — wire locale provider:

    The SDK module needs to pass a localeProvider that reads the current app locale. Since SdkModule is in core/sdk (common module), it cannot directly import composeApp's getAppLocale(). Instead, pass it as a factory function from the DI graph.

    Add to sdkModule:
    ```kotlin
    single {
        createApiClient(
            authInterceptor = get(),
            baseUrl = defaultBaseUrl(),
            localeProvider = getOrNull<() -> String>(named("localeProvider")) ?: { "en" },
        )
    }
    ```

    Then in the composeApp module's Koin setup (wherever allAppModules is assembled), add:
    ```kotlin
    single<() -> String>(named("localeProvider")) { { getAppLocale() } }
    ```

    ALTERNATIVELY (simpler approach): Since PreferencesStorage is already in the DI graph and has the `language` property, use that:
    ```kotlin
    single {
        val storage: PreferencesStorage = get()
        createApiClient(
            authInterceptor = get(),
            baseUrl = defaultBaseUrl(),
            localeProvider = { storage.language },
        )
    }
    ```

    This is simpler — PreferencesStorage is already available in the DI graph from storageModule. Use this approach.

    Add import for PreferencesStorage:
    ```kotlin
    import com.m2f.template.storage.PreferencesStorage
    ```

    NOTE: Verify PreferencesStorage is accessible from core/sdk module. Check if core/sdk depends on core/storage in the Gradle build. If NOT, you'll need the named("localeProvider") approach instead. Check `core/sdk/build.gradle.kts` for dependencies.

    3. MODIFY Application.kt — allow Accept-Language in CORS:

    In the CORS block, add:

    The server-side ServerStrings.resolve() already reads Accept-Language from Ktor's request headers — it just never receives it because the client doesn't send it and CORS blocks it.

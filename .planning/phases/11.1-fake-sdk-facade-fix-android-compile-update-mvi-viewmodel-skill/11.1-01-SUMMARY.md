## Decisions Made
- Used direct Sdk constructor instantiation instead of subclassing (plan suggested FakeSdkImpl extending Sdk, but Sdk is a final class). Since Sdk already delegates to its constructor params via `AuthApi by authApi, UserApi by userApi`, passing fake APIs to the constructor achieves the same result without needing inheritance.
- Android kotlin-test-junit must be declared independently of jvmMain because androidTarget is a separate KMP compilation unit that does not inherit jvmMain dependencies.

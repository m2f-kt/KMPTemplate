**Step 3: Create MviViewModel.kt**

Create `core/mvi/src/commonMain/kotlin/com/m2f/template/core/mvi/MviViewModel.kt` with the exact reference implementation from RESEARCH.md, with one addition: set `extraBufferCapacity = 64` on the pipeline MutableSharedFlow to prevent emit() from suspending before collectors are ready.

Key API requirements (all from locked decisions):
- `abstract class MviViewModel<Intent, Model, Mutation, Event>` extends `ViewModel()`
- Constructor: `initialState: Model`, `modelSharingStarted: SharingStarted = SharingStarted.WhileSubscribed(5_000)`
- Private pipeline: `MutableSharedFlow<Either<Event, Mutation>>(extraBufferCapacity = 64)`
- Public `val model: StateFlow<Model> by lazy { ... }` using filterIsInstance + scan + stateIn
- Public `val event: SharedFlow<Event> by lazy { ... }` using filterIsInstance + map + shareIn
- Abstract `fun take(intent: Intent)` (not suspend)
- Abstract `suspend fun reduce(model: Model, mutation: Mutation): Model` (suspend per user decision)
- Protected `suspend fun sendEvent(event: Event)`
- Protected `suspend fun sendMutation(mutation: Mutation)`
- Protected `suspend fun sendStatement(statement: Either<Event, Mutation>)`

No extra convenience methods beyond the reference. No demo ViewModel. No Koin module.

    3. **ApiClient.kt** (commonMain): Create `createApiClient` function:
       ```kotlin
       package com.m2f.template.sdk

       fun createApiClient(
           baseUrl: String = "http://localhost:8080",
           tokenProvider: (() -> String?)? = null,
       ): HttpClient {
           return HttpClient(platformEngine()) {
               expectSuccess = false  // CRITICAL: auth interceptor needs to see 401s

                   json(Json {
                       ignoreUnknownKeys = true
                       isLenient = true

           }
       }
       ```

       - The `tokenProvider` parameter is nullable and unused in this plan -- Plan 03-03 will add the AuthInterceptor that uses it.
       - Do NOT install the Ktor Auth plugin (WasmJs incompatible per research).

    4. **ErrorMapper.kt** (commonMain): Create the core error mapping utilities:
       ```kotlin
       package com.m2f.template.sdk

       suspend inline fun <reified T> apiCall(
           block: () -> HttpResponse
       ): Either<AppError, T>
       ```
       Implementation:
       - Wrap `block()` in try/catch
       - On success (`response.status.isSuccess()`): deserialize body to `T` via `response.body<T>()` and return `Right(T)`
       - On HTTP error: call `mapHttpError(response)` and return `Left(AppError)`
       - On exception: call `mapException(e)` and return `Left(AppError)`

       `mapHttpError` implementation per research Pattern 3:
       - 401 -> `AppError.Auth.Unauthorized(message = errorResponse?.message ?: "Authentication required")`
       - 403 -> `AppError.User.Forbidden(message = errorResponse?.message ?: "Access denied")`
       - 404 -> `AppError.User.NotFound(message = errorResponse?.message ?: "Not found")`
       - 409 -> `AppError.Auth.UserAlreadyExists(message = errorResponse?.message ?: "Resource already exists")`
       - 422 -> `AppError.Client.ServerMapped(code = errorResponse?.code ?: "VALIDATION_ERROR", message = errorResponse?.message ?: "Validation failed")`
       - 500..599 -> `AppError.Server.Internal(message = errorResponse?.message ?: "Server error")`
       - else -> `AppError.Client.Unknown(detail = errorResponse?.message ?: "HTTP ${response.status.value}")`

       `mapException` implementation:
       - `is IOException` -> `AppError.Client.Network()`
       - `is kotlinx.coroutines.TimeoutCancellationException` -> `AppError.Client.Timeout()`
       - else -> `AppError.Client.Unknown(detail = e.message)`

       Note: Use `kotlin.io.IOException` for multiplatform IOException (available in Kotlin 2.x stdlib). If that doesn't exist on all targets, catch `Exception` generally and check for network-specific names. The safest cross-platform approach: catch `Exception` and use `AppError.Client.Unknown(detail = e.message)` for unrecognized exceptions.

    5. **Sdk.kt**: Delete the stub file content (just has package declaration). Or repurpose it as a convenience facade if desired. Simplest: leave it empty or delete it.

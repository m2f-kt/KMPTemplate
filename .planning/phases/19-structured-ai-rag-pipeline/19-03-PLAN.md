       - `suspend fun findById(id: Uuid): DocumentRow?` -- SELECT by id, return data class `DocumentRow` with all columns

       - `suspend fun delete(id: Uuid): Boolean` -- DELETE by id, return true if affected > 0

    2. Create `DocumentIngestionService` in `server/ai/rag/`:
       - Constructor: `LLMEmbedder` (Koog embedder with text-embedding-004), `R2dbcDatabase`, `DocumentRepository`, `CoroutineDispatcher` (aiDispatcher for embedding calls)
       - `suspend fun ingest(documentId: Uuid, groupId: Uuid, userId: Uuid, text: String)`:
         a. Update document status to "processing"
         b. Chunk text using simple character-based chunking: ~500 chars per chunk with ~100 char overlap (per research -- yields ~125 tokens, well under 2048 limit)
         c. For each chunk, embed via `embedder.embed(chunk)` which returns `Vector(values: List<Double>)` with 768 dimensions
         d. Store each embedding in document_embeddings via raw SQL INSERT (not through PgVectorStorage factory since we need documentId/chunkIndex):
            - Use `suspendTransaction(db = db)` with DocumentEmbeddingsTable INSERT
            - Set groupId, userId, content (chunk text), embedding (convert Double->Float), documentId, chunkIndex
         e. Update document status to "indexed" with chunkCount
         f. On any error: update status to "failed", log error, do NOT re-raise (don't break the upload response)
       - `fun chunkText(text: String, chunkSize: Int = 500, overlap: Int = 100): List<String>` -- private helper for character-based chunking
       - `suspend fun deleteEmbeddings(documentId: Uuid)` -- DELETE FROM document_embeddings WHERE document_id = documentId (via PgVectorStorage.deleteByDocumentId or direct SQL)
       - Auto-ingest on upload per Claude's discretion (user said "Claude's discretion on auto-ingest vs explicit").

    3. Add document-specific DomainError subclasses to `AiErrors.kt`:
       - `class DocumentNotFound : DomainError()` with 404 respond
       - `class DocumentAccessDenied : DomainError()` with 403 respond
       - `class DocumentIngestionFailed(val detail: String) : DomainError()` with 500 respond
       Follow the existing AiErrors pattern (ProviderUnavailable, AgentExecutionFailed).

    Supported file types (Claude's discretion): `.txt` and `.md` for MVP. PDF can be added later per research recommendation. Extract text directly from uploaded bytes (UTF-8 decode).

  <action>
    Create `PgVectorStorage` implementing Koog's `VectorStorage<String>` in `server/ai/rag/`:

    Constructor parameters:
    - `db: R2dbcDatabase`
    - `groupId: Uuid` -- scoping for group isolation (RAG-06)
    - `userId: Uuid?` -- null for admin (sees group + own), non-null for member (sees own only)
    - `isAdmin: Boolean` -- controls scope logic

    Implement all `VectorStorage<String>` interface methods:
    - `store(document: String, data: Vector): String` -- INSERT into document_embeddings with groupId, content, embedding (convert `Vector.values` from `List<Double>` to `List<Float>` at the storage boundary per research), userId, documentId (set externally or null). Return generated UUID as string.
    - `delete(documentId: String): Boolean` -- DELETE by id, return true if rows affected > 0
    - `read(documentId: String): String?` -- SELECT content by id
    - `getPayload(documentId: String): Vector?` -- SELECT embedding by id, convert `List<Float>` back to `Vector(List<Double>)`
    - `readWithPayload(documentId: String): DocumentWithPayload<String, Vector>?` -- combine read + getPayload
    - `allDocuments(): Flow<String>` -- SELECT all content for this group scope (use flow builder with suspendTransaction)
    - `allDocumentsWithPayload(): Flow<DocumentWithPayload<String, Vector>>` -- SELECT all with embeddings

    Add custom method (NOT part of VectorStorage interface):
    - `suspend fun searchSimilar(queryVector: Vector, topK: Int = 5, minSimilarity: Double = 0.0): List<SimilarChunk>` where `SimilarChunk` is a local data class with `content: String, similarity: Double, documentId: String?`
    - Use raw SQL via `TransactionManager.current().exec()` with pgvector `<=>` cosine distance operator:
      ```sql
      SELECT id, content, document_id, 1 - (embedding <=> ?::vector) as similarity
      FROM document_embeddings
      WHERE group_id = ?
      ${scopeClause()}
      AND 1 - (embedding <=> ?::vector) >= ?
      ORDER BY embedding <=> ?::vector
      LIMIT ?
      ```
    - `scopeClause()` returns: for members -> `AND user_id = '$userId'`; for admins -> empty string (sees all within group)
    - IMPORTANT: Do NOT use `EmbeddingBasedDocumentStorage` for retrieval (loads all into memory per research pitfall #3). Use pgvector `<=>` operator for server-side similarity.

    Add helper method:
    - `suspend fun deleteByDocumentId(docId: Uuid)` -- DELETE all embeddings WHERE document_id = docId (for document deletion cleanup per user decision)

    Use `suspendTransaction(db = db)` for all database operations. Follow the existing ExposedPersistenceStorage pattern.
